### 参考库调用铁律

1. **内嵌保底 + 参考深入**：SKILL.md 每个 Phase 内嵌的「核心技法速查」是保底规则，**无论参考文件是否加载都必须遵守**。参考文件提供扩展讨论、更多案例和深层原理，在上下文充裕或需要深入讨论时按需加载。
2. **阶段必读不可跳过**：进入每个 Phase 前，必须读取下方矩阵中的必读参考；快速模式只跳过长讨论，不跳过内部加载。
3. **输出必须带调用痕迹**：大纲、细纲、正文、润色和审核摘要中必须列出本步读取的参考库与实际采用的技法。
4. **文风圣经长期生效**：Phase 4 生成或更新 `memory/project_style.md` 后，Phase 6、8、10、12 必须优先读取，不得每章重新临时选风格。若项目启用文风F（对比学习文风），则 `memory/style_contrast_guide.md` 与 `memory/project_style.md` 同步强制读取，对比规则优先级高于泛化去AI味规则。
5. **缺必读参考即阻断**：必读参考文件缺失时，先提示缺失项并停止该阶段；不得用泛化经验替代。
6. **读取确认硬门控**：Phase 6/7/8/10 生成任何内容前，必须先输出「读取确认块」，列出已读文件路径 + 从每个文件中提取的 1-2 条关键规则/事实（直接引用或精准复述）。未输出该块 → 阻断生成，提示"请先完成读取确认"。读取确认块格式见各 Phase 的 DP{N}-0 决策点。
7. **读取顺序**：项目约束 > 真相档案 > 参考库。即 `memory/project_style.md`（若文风F则加 `memory/style_contrast_guide.md`）→ `.learnings/` 活跃档案 → `references/` 必读参考。项目级约束优先于通用技法，避免被泛化经验覆盖。
8. **读取范围补全**：Phase 6/7/8 在原有 references/ 必读基础上，额外必读 `.learnings/` 活跃档案（至少 `STORY_BIBLE.md`、`CHARACTERS.md`、`PLOT_SUSPENSE.md`；涉及地点/资源/支线/情感时同步读取对应档案）。新书首次生成大纲且无已写正文时，可标注"无已写正文"跳过该层。Continuation_Mode 下 Phase 7-9 额外必读 `设定/不可改动清单.md`。

## 核心哲学

1. **大纲是地图，不是牢笼** — 粗到能看见全局，细到能坐下就写
2. **爽点密度决定存亡** — 每3000-5000字必须有一个让读者"爽"的情绪节点
3. **日更是底线，质量是上限** — 稳定产出比单章完美更重要
4. **先写30章，再谈其他的** — 大纲不需要完美，写出来自然发现方向
5. **网文写作是工程，不是灵感** — 靠工程可以写200万字，靠灵感不行

### 全流程交互模式

创作流程提供两种交互模式，用户可随时切换：

| 模式 | 特点 | 适合用户 |
|------|------|---------|
| 🗨️ 讨论模式（默认） | 每个决策点都展示选项+讨论 | 想深度参与方向的作者 |
| ⚡ 快速模式 | 只在关键决策点暂停，其余自动推进 | 想快速出稿的作者 |

**切换方式**：用户说"切换到快速模式"/"切换到讨论模式"

**快速模式必停点**：DP6-0（大纲读取确认，硬门控）、DP7-0（细纲规划读取确认，硬门控）、DP8-0（生成细纲读取确认，硬门控）、DP10-1（正文读取确认，硬门控）、DP1-2（方案选择）、DP4-1（参数配置）、DP9-1（审核方案选择）、DP10-2（正文审核，首章）

**决策点编号体系**：主流程使用 `DP` 前缀（DP1-1~DP12-3），续写流程使用 `CP-` 前缀（CP-①~CP-⑫）。注意区分：CP1-CP3 是续写**准备阶段**（下表），CP-①~CP-⑫ 是贯穿续写全流程的12个**交互决策点**。

> 完整决策点交互模板见 `references/workflow-interaction.md`，提示词模板见 `references/interactive-prompts.md`。

## Phase 1：灵感生成

**目标**：确定小说基本方向，生成3套完整创意方案

### 提示词自动完善（8维度）

按以下维度自动补全用户输入：①题材定位 ②世界观设定 ③主角人设 ④核心冲突 ⑤爽点设计 ⑥节奏规划 ⑦配角框架 ⑧开篇钩子。完整模板见 `assets/PROMPT-TEMPLATE.md`。

### 快速问卷（可跳过）

如用户方向模糊：①题材 ②核心卖点 ③目标字数 ④对标书

### 对标上下文加载

如用户提到对标书或 `对标/` 目录存在：检查 `拆文报告.md`，有则读取，无则提示是否先做拆文分析。

### 输出

1. **3个备选书名** 2. **一句话简介**（主角+困境+金手指+目标） 3. **核心冲突** 4. **核心爽点**

> 决策点：DP1-1 创作方向确认 → DP1-2 创意方案选择。详见 `references/workflow-interaction.md`。

## Phase 2：世界观构建

**目标**：基于选定题材构建完整、自洽的世界观

确定4要素：①时代背景 ②力量体系 ③社会结构 ④核心规则

设计要点：力量体系须有边界和代价 / 世界观展开跟剧情推进绑定 / 日常细节比宏大设定更让世界"活" / 每5章须有世界观新信息

> 详细世界观构建模板见 `references/style-fantasy.md`。

### 输出文件

- `设定/世界观/背景设定.md`：时代背景与核心规则
- `设定/世界观/力量体系.md`：力量等级与边界
- `设定/题材定位.md`：题材定位与核心卖点

> 决策点：DP2-1 世界观方向选择 → DP2-2 世界观细节审核。详见 `references/workflow-interaction.md`。

**目标**：创建立体、有深度的角色

### 大纲创建五步法

1. **确定高潮剧情** — 最大冲突规模、最多出场人物、最强情绪波动
2. **确定单元剧** — 金手指获得爽点，相邻单元剧不能用相同逻辑
3. **故事线组织** — 同时管理8条线（地图/阵营/人物/金手指/世界观/矛盾/收集/感情线）
4. **确定开篇舞台** — 情节钩子→异常→金手指激活→矛盾→目标
5. **确定收尾** — 解决主线+额外奖励+角色结局+下卷铺垫

### 关键节点（100章基准，按比例缩放）

| 节点 | 比例 | 内容 |
|------|------|------|
| 开篇钩子 | 1% | 必须吸引读者 |
| 第一个高潮 | 10% | 确立故事方向 |
| 中点转折 | 50% | 提升故事格局 |
| 至暗时刻 | 65% | 情感最低点 |
| 最终高潮 | 85% | 所有矛盾爆发 |

> 完整大纲技法（三幕式+升级感设计+矛盾设计+卷级大纲模板）见 `references/outline-arrangement.md` 和 `references/plot-structures.md`。

### 核心技法速查（大纲必遵）

**结构设计**：
1. **高潮倒推法**：先定全书最大高潮，反向推导需要什么伏笔/角色/能力，再正向铺路。高潮决定一切前置条件。
2. **八节点骨架**：开篇钩子(1%)→第一高潮(10%)→升级转折(25%)→中点转折(50%)→虚假胜利(60%)→至暗时刻(65%)→最终高潮(85%)→收尾(95%)。
3. **AB交织**：A线（主线矛盾）和B线（感情/成长/世界观）交替推进，A线紧张时B线给喘息，B线深入时A线埋伏笔。
4. **卷级升级感**：每卷必须有一种"升级"——实力/地图/认知/关系/格局。卷末状态必须高于卷首，读者才能感受到前进。

**爽点与期待感**：
5. **两长一短**：始终保持三个开放问题——两个长线目标（全书/卷级）+ 一个短线目标（当前批次）。长线不能断，短线快速兑现。
6. **钩子链**：一个期待兑现之前，必须先挂上下一个钩子。升级完立刻出新威胁，不给"目标真空"。
7. **杠杆拉扯法**：确立打脸目标后，做 6 轮"旁人质疑 + 主角暗中准备 + 读者悲观"，最终打脸时爽感翻倍。
8. **反派五要素**：人设 + 动机 + 具体恶行 + 态度（越"正义"越让人恨）+ 对主角的伤害程度。反派智商/实力决定主角含金量。
9. **打脸三段式**：反派 20% 嚣张铺垫 → 60% 反派横行/主角隐忍 → 20% 主角出手翻盘。

**伏笔与信息管理**：
10. **伏笔分级**：短线伏笔（3-5章回收）、中线（一卷内）、长线（跨卷）。每条伏笔必须记录回收计划。
11. **信息差武器**：读者知道而角色不知道 = 悬念；角色知道而读者不知道 = 反转；双方都不知道 = 探索。三种信息差交替使用。
12. **断期待六禁忌**：目标真空（主角无事可做）、角色真空（配角消失）、地图真空（场景停滞）、支线泛滥（主线被淹）、目标模糊、升级无差异化。

> 更多技法（八线管理细节、矛盾设计公式、情绪模块组合）见参考文件。

5. **跨批缓冲**：生成当前批次时，同步规划下一批的"开头方向"（只定方向不定细纲），当前批次末尾钩子必须指向该方向。

**爽点类型轮换**：打脸型 / 突破型 / 揭秘型 / 获宝型 / 关系升温型 / 逆转绝境型。**同类爽点不得连续 3 章使用**，必须轮换。

**新概念铺垫链标注**：本章若有新机制/新设定/新术语首次登场，必须在章节细纲中标注三阶铺垫链分布——①微伏笔（感官暗示，位置+细节）②逻辑桥梁（认知推导，方式+框架）③体感落地（完整体验，触发→运转→结果）。三阶缺任一阶→阻断，参见 `pave-chain-technique` 技能。跨章分布时每章标注负责哪一阶。

**每章功能检查**：删除本章后读者是否会察觉缺失？不会 = 水文，必须赋予功能（推进/揭示/伏笔/关系变化至少占一项）。

## Phase 9：逻辑审核与人工介入

**目标**：审核当前批次，给出可选方案，让用户介入后决定冻结/小修/回退

### 审核维度

| 维度 | 检查项 |
|------|--------|
| 时间线 | 章节衔接、时间跳跃、成长速度 |
| 空间线 | 地点转移、行动距离、地图切换成本 |
| 人物线 | 存活状态、等级匹配、关系变化、动机连贯 |
| 设定线 | 力量体系、物品来源、社会规则、系统边界 |
| 伏笔线 | 埋设、回收时机、后续可承接性 |
| 读感线 | 章节功能、爽点兑现、章末钩子有效性 |
| 技法调用完整性 | 是否使用必读参考库、是否符合 `memory/project_style.md`、是否有爽点/钩子/节奏来源 |

### 审核流程

1. 读取上一批已冻结细纲+当前批次细纲+当前卷目标+本批参考调用清单 2. 逐章+跨章节检查 3. 检查爽点库、钩子库、剧情技法库和大神参考是否真正落入细纲 4. 记录问题并判断级别（微调/重做/回改） 5. 生成2-4个方案+自定义选项，进入人工介入 6. 用户选择后修改并重新审核，直到满足冻结条件

### 冻结条件

无硬逻辑矛盾 / 本批次承诺的爽点已兑现 / 本批次结尾能自然牵引下一批 / 必读参考调用完整 / 人工介入节点已完成

### 核心审核清单（逻辑审核必遵）

**时间线**：
- 事件时序自洽？（无"先果后因"，倒叙有标注）
- 爽点间隔不超 5000 字，虐点不密集无喘息？

**空间线**：
- 每个场景有明确的目标、阻碍与变化？（场景结束后局势不同于开始前）
- 场景描写超 500 字却未推进剧情 = 水字数？

**人物线**：
- 主角行为符合已建立的人设？情绪通过动作落地而非直述？
- 不同角色对话方式有区分度？配角有存在感，反派逼格匹配当前阶段？

**设定线**：
- 战力/经济/世界观设定前后一致？（无崩坏，以普通人为锚点）
- 金手指/核心能力的效果展示清晰完整？

**伏笔线**：
- 前文埋下的伏笔/承诺已有推进或回收？（钩子不回收 = 烂尾）
- 证据/线索分次释放而非一次性倾泻？（至少分 3 次揭露）

**读感线**：
- 开头前 50 字有冲突或异常？前 500 字有钩子？
- 结尾落在变化上（动作/对话/画面），而非总结或反思？
- 存在可整段删除而不影响理解的内容 = 水文？

**技法调用完整性**：
- 文风是否偏离 `memory/project_style.md`？爽点是否兑现？章末钩子是否有效？是否违背大神参考节奏？

**铺垫链完整性**：
- 本批次是否有新概念/新机制首次登场？如有，三阶（微伏笔→逻辑桥梁→体感落地）是否完整？
- 三阶间距是否合理？（微伏笔→逻辑桥梁 ≤2 章，逻辑桥梁→体感落地 ≤1 章，全程 ≤3 章）
- 是否存在违规模式？（面板倾泻 / 导师说教 / 无感触发 / 有伏笔无落地 / 三阶乱序）

### 开头设计（首章必遵）

**黄金一章法则**：从故事最精彩、最有冲突的地方写起，不是从"背景介绍"写起。

1. **300 字规则**：主角必须在 300 字内出场，1000 字内出现一个爽点或期待点。
2. **开头信息释放顺序**：危机 > 角色 > 金手指暗示 > 世界观。每次释放必须服务当前情绪目标。
3. **开头禁令**：不写回忆录、不切视角、不写梦境、不写长段对话。前两章以叙事为主，压缩描写和说明。
4. **三种开局模板**：困境切入（主角劣势→压迫→金手指激活→部分解决但更大问题浮现）、优势切入（隐藏实力→意外暴露→旁人认知崩塌→暴露引来更大麻烦）、悬念切入（反常事件→读者好奇→逐步揭示→引出主线矛盾）。
5. **金手指时机**：系统流必须第 1 章展示金手指；其他题材可延后，但延后期间必须有足够强的替代吸引力。

> 题材开头数据库（都市/仙侠/甜宠/宅斗/校园等 8 种题材专属开头模式）见 `references/opening-design.md`。

### 写作核心技法（正文每章必遵）

**叙事四铁律**：
1. **"然后呢"原则**：每段话都要回答读者心中的"然后呢"。写完一个信息点，立刻用下一个接上。没有"死段落"——既不推进剧情也不揭示角色的段落必须删掉。
2. **间接描写**：不直接写场面多爆炸，写旁观者的反应。正面描写是铺垫，侧面反应才是爽点。"他很强" → 不如写围观者的震惊表情。
3. **白描精准**：用最少的字传达最准的信息。精准的动词和名词 > 形容词堆砌。"怒视"比"很生气地看着"有力十倍。
4. **一笔多用**：每段至少承载 2 个功能（环境+伏笔 / 对话+角色塑造 / 动作+世界观展示）。单功能段落是水文。

**节奏控制**：
5. **句式交替**：三句短句后接一句长句，反之亦然。紧张场景用 3-8 字短句，日常场景用 8-15 字中句，描写用不超过 20 字。
6. **镜头思维**：每段 = 一个镜头。远景（环境）→ 中景（人物关系）→ 特写（细节/情绪）。快节奏 = 短切镜；关键时刻 = 慢动作特写。
7. **场景过渡用感官桥**：通过共享物件/感觉/情绪连接场景。时间跳转："放下茶杯 → 第二天清晨"；空间跳转："远处钟声 → 另一座城的黄昏"。

**情绪与代入感**：
8. **情弦理论**：动笔前确定目标读者的核心情绪需求（被认可/复仇/恋爱/升级），所有情节围绕这根弦演奏。
9. **情绪表达层级**：最低级→"他很愤怒"（直说）；中级→"脸色铁青"（表情）；最高级→"一拳砸在桌上，指甲嵌进掌心"（动作）。永远用动作/行为展示情绪，不用标签。
10. **五感描写**：每个场景至少调动 2-3 种感官（视/听/触/嗅/味）。五感必须服务情绪：恐惧写冷、暗、静；兴奋写亮、快、响。

**对话技法**：
11. **每句对话必须有功能**：推进剧情（揭示新信息）/ 构建期待（暗示将发生什么）/ 渲染情绪（角色语气传达状态）。三者皆无 = 废话，删掉。
12. **角色声音分化**：每个角色配：口头禅/口癖、语速节奏（啰嗦 vs 干脆）、领域词汇（行话 vs 俗语）、身份语域（长辈/晚辈/贵族/平民）。
13. **潜台词设计**：角色真正动机永远不直说——用借口和合理化包裹。好对话 = 两个冲突的议程在试探、博弈、碰撞。

**水文检测**：删除本段后读者不会察觉缺失 = 水文。连续 2 章以上无冲突 = 水文。同一种情绪写 3 段以上 = 水文。场景描写超 500 字不推进剧情 = 水文。

**新概念铺垫链执行**（新概念首次登场必遵）：
1. **微伏笔**：在正式登场前植入 1-3 句感官异常细节，异常但不解释，与后续机制有因果关联
2. **逻辑桥梁**：通过角色内心独白/对话/观察建立认知框架，用已知经验推导未知概念，可留 20-30% 疑问
3. **体感落地**：完整描写机制运作（触发→运转→结果→回味），五感贯穿全程，面板提示融入叙事比喻体系
4. 三阶顺序不可乱，间距不可超过 3 章。每完成一个概念的三阶，在章节摘要中记录：`[概念名称] 铺垫链完成：微伏笔(L##) → 逻辑桥梁(L##) → 体感落地(L##)`

> 详细规则、正反案例和门控校验见 `pave-chain-technique` 技能。

> 更多技法（环境交互规则、核心梗迭代法、阵营 staging 法）见 `references/writing-basics.md` 和 `references/writing-craft.md`。

### 去AI味写作规范（正文必遵）

**六门禁**：
1. **段落≤4行**：超过就拆——用对话、动作节拍或更短的观察打断。
2. **动作+对话+情绪循环**：不写超过 2 段连续纯内心独白。
3. **情绪用身体写**：禁止"他很紧张" → "手心冒汗，筷子差点没夹住"。禁止"她愤怒" → "她把杯子摔了，碎片溅到脚踝也没弯腰捡"。
4. **口语化**：禁止"我认为此事不妥" → "我觉得不靠谱"。成语 → 大白话。"目光如炬" → "眼珠子一动不动盯着"。
5. **章末禁令**：不用总结性感悟（"他终于明白了……"）、不用哲学感叹（"人生就是这样……"）、不用预告式悬念（"他不知道更大的风暴正在来临"）。用动作、对话或未解决的紧张收束。
6. **去欧化**：禁止"很/非常"堆叠、禁止被动"被"滥用（"他被激怒了" → "他火了"）、禁止"作出决定/进行考虑" → 直接"决定/考虑"、禁止翻译腔（"名为……的……""令人……""最……之一"）。

**频率控制**（每章自检）：
- 弱化副词（微微/淡淡/缓缓/轻轻/不禁/不由自主）：每千字不超过 2 次
- 情绪标签词（紧张/愤怒/恐惧等直接命名情绪的词汇）：每章不超过 1 次
- 比喻词（像/仿佛/宛如/犹如）：每章不超过 1 次，且必须功能性使用（服务于剧情而非装饰）
- 禁语表高频词（冷笑/颤抖/泛白/勾起弧度/空气凝固/心中暗道/映入眼帘）：每章各不超过 1 次

**三遍法润色**（写完后执行）：
- 第一遍·去泛化：删抽象总结、假深度、膨胀意义、装饰性形容词 → 消除 80% AI 味
- 第二遍·去书面：把分析/公文/学术词汇替换为日常口语
- 第三遍·回人味：加入具体感官细节、区分角色声音、打散节奏

**自检口诀**："一段不过四行字。对话要像人说话。心情不写心里话。结尾不搞大升华。打斗不写流水账。日常要埋伏笔桩。"

> 完整禁语表、Show Don't Tell 对照表、七种 AI 模式详细识别法见 `references/anti-ai-writing.md`。

> 详细写入规范见 `references/artifact-protocols.md`

**必写**：`.learnings/PLOT_SUSPENSE.md`（关键情节+伏笔）、`追踪/CHAPTERS.md`（章节索引）、`SESSION.md`（会话状态）

**有变化则写**：`.learnings/CHARACTERS.md`（角色变化）、`.learnings/LOCATIONS.md`（新地点）、`.learnings/RESOURCES.md`（物品/金钱）、`.learnings/SUBPLOTS.md`（子剧情）、`.learnings/EMOTIONS.md`（情感发展）

**Continuation_Mode 下额外规则**：每条写入项末尾追加 metadata 标注来源为 `AI续写`，`追踪/CHAPTERS.md` 追加来源标记列区分原作/续写章节。

> 决策点：DP10-1 章节生成前确认（硬门控，必停） → DP10-2 章节生成后审核（首章必停）。详见 `references/workflow-interaction.md`。

## Phase 11：一致性审核

**目标**：检查正文与设定、前文的一致性（剧情/角色/设定层面）

### 审核前读取

必须读取 `quality-check.md`、`quality-checklist.md`、当前章节正文、当前批次细纲、`memory/project_style.md` 和本章参考调用清单。审核维度中必须包含“技法调用完整性”：文风是否偏离、爽点是否兑现、章末钩子是否有效、是否违背大神参考节奏。

检查维度：核心一致性（剧情/人设/设定前后矛盾） / 格式一致性（对话格式/场景切换/时间线） / 逻辑连贯（动机/因果链/时间线）

### 核心审核清单（一致性审核必遵）

**核心一致性（剧情/人设/设定）**：
- 角色等级、状态、能力与 `CHARACTERS.md` 记录一致？无已死/已退场角色意外出场？
- 地点、势力、世界观设定与 `LOCATIONS.md` / `STORY_BIBLE.md` 无矛盾？
- 本章剧情承接上章结尾，无断链或重复事件？

**格式一致性**：
- 对话标记格式全书统一（引号样式、说话人标注方式）？
- 场景切换有明确分隔符或过渡句，且全书风格一致？
- 时间线表述前后一致（"三天后"与前文日期吻合）？

**逻辑连贯（动机/因果链）**：
- 角色每个决定符合其已建立的性格与动机，无突兀行为？
- 事件因果链完整——前因在本章或前文有交代，后果不凭空出现？
- 反派有合理智商与动机，而非纯粹"送人头"？

**铺垫链完整性**：
- 本章新概念首次登场是否三阶完整（微伏笔→逻辑桥梁→体感落地）？间距是否合理？
- 有无面板倾泻、导师说教、无感触发等违规模式？

**技法调用完整性**：
- 去AI味八项检查全部执行？
- 爽点结构完整（铺垫→冲突升级→高潮爆发→章末钩子）？
- 各角色对话风格有区分度，且与前期章节保持一致？

> 完整审核维度和评分体系见 `references/quality-check.md`，检查清单见 `references/quality-checklist.md`。

#### Continuation_Mode 下的额外检查：风格指纹比对

计算当前续写章节相对于 `设定/风格指纹.md` 的9项指标偏离度，产出 `追踪/续写风格对比_第{N}章.md`。3项以上指标超过漂移阈值 → 阻断 Phase 12，用户决定"重写/调整阈值/接受偏离"。

> 详细风格指纹比对规则见 `references/style-fingerprint-guide.md`。

> 决策点：DP11-1 一致性审核讨论。详见 `references/workflow-interaction.md`。

## Phase 12：润色与评分

**目标**：对正文做最终润色（语言质量层面），确保质量达标

> Phase 11 负责"一致性"，Phase 12 负责"语言质量"（去AI味/润色/评分）。

### 润色前读取

必须读取 `memory/project_style.md`（若文风F启用则同步读取 `memory/style_contrast_guide.md`）、`quality-check.md`、`quality-checklist.md`、`anti-ai-writing.md`；关键高潮章、卷终章或低分章节额外读取 `advanced-audit.md`。润色摘要必须标注本步应用了哪些文风/大神技法回拉规则，以及对比文风规则（若F启用）的回检结果。

**续写/正文中途激活文风F**：若 Phase 12 润色中用户要求"用F文风"/提供对比样本，立即暂停润色 → 执行对比分析（同 CP-⑦½ 流程）→ 写入 `memory/style_contrast_guide.md` → 回流继续润色，对比规则即时生效。当前章节按新规则重新润色。

### 去AI味检查项（润色必过）

**8 项逐条检查**：
1. 无比喻词超频（每章 ≤1 次，且必须功能性使用）
2. 无直白心理描写（"他很紧张/她感到愤怒" → 全部改为动作外化）
3. 禁语表词汇未超频（弱化副词每千字 ≤2 次，情绪标签词每章 ≤1 次）
4. 段落 ≤4 行
5. 章尾无总结/升华/预告
6. 无"被"字滥用和欧化表达（"被激怒" → "火了"，"作出决定" → "决定"）
7. 无 AI 套话（"映入眼帘/心中暗道/意义深远/未来可期/综上所述/由此可见"）
8. 对话像人说话（口语化、有角色声音分化、无书面语混入）

**三遍法执行**：第一遍去泛化（删抽象总结和装饰性形容词）→ 第二遍去书面（替换公文/学术词汇为口语）→ 第三遍回人味（加感官细节、区分角色声音、打散节奏）

> 完整禁语表、Show Don't Tell 对照表、AI 指纹识别方法见 `references/anti-ai-writing.md`。

#### Continuation_Mode 下的额外润色：风格指纹回拉

润色阶段额外执行风格指纹回拉规则，把超阈值的句长、段长、对话占比、高频意象重新拉向原作基线。润色后重新计算9项指标，在 `追踪/续写风格对比_第{N}章.md` 追加"润色后"列。

### 质量检查（7 维度评分）

**评分维度与权重**：爽点设置(25%) / 情节连贯性(20%) / 节奏把控(15%) / 悬念钩子(15%) / 对话质量(10%) / 角色塑造(8%) / 去AI味(7%)

**每维度自检问题**：
- **爽点(25%)**：主角做了什么让读者"爽"？被打脸的反派反应够夸张吗？围观群众的震惊写到位了吗？
- **连贯(20%)**：角色实力/位置/状态匹配角色卡和地点卡？有没有违背世界观规则？接上了上一章结尾？
- **节奏(15%)**：本章 200 字内进入主要矛盾了吗？冲突是逐步升级而非一次性倾倒？高潮有足够的铺垫？
- **钩子(15%)**：章末留下了什么问题？是"发生了什么"（较弱）还是"为什么会这样"（较强）？这个钩子会让你点"下一章"吗？
- **对话(10%)**：主角有独特的说话风格吗？反派够嚣张吗？配角各有辨识度吗？每句对话都推进剧情了吗？
- **角色(8%)**：角色行为符合动机链吗？情绪变化有触发事件吗？有成长/变化吗？
- **去AI味(7%)**：过 Phase 12 去AI味 8 项检查了吗？

**等级对照**：S(9.0+) = 精品 / A(8.0-8.9) = 优质 / B(7.0-7.9) = 可出稿 / C(6.0-6.9) = 需修改 / D(<6.0) = 重写

> 完整评分细则和示例见 `references/quality-check.md`，检查清单见 `references/quality-checklist.md`。

### 出稿门槛

| 场景 | 最低评分 |
|------|---------|
| 普通日更 | 总分 >= 7.0 |
| 关键高潮章 | 总分 >= 8.0 |
| 卷终章 | 总分 >= 8.2 |
| 续写章节（Continuation_Mode） | 总分 >= 7.5 |
| 发布前精修 | 必跑深度审稿 |

### 低分回退机制

当章节评分低于出稿门槛时，按以下规则回退：

| 低分维度 | 回退目标 | 处理方式 |
|---------|---------|---------|
| 节奏/爽点密度不足 | Phase 8 细纲 | 重新规划本章爽点分布和节奏曲线 |
| 人设崩塌/对话OOC | Phase 3 角色设定 | 补充角色心理动机和语言特征，重写对话 |
| 剧情逻辑漏洞 | Phase 6 大纲 | 修补逻辑链，必要时调整前后章节 |
| 文笔AI味重 | Phase 12 润色 | 强化去AI味检查，参考反AI味写作指南 |
| 总分 < 6.0 | Phase 8 细纲 | 整体重写本章，不尝试修补 |

**回退次数限制**：同一章节最多回退 2 次。第 3 次仍低于门槛时，暂停并提示用户介入（调整预期、简化剧情、或跳过本章）。

### 记忆写入（润色评分后必执行）

| 文件 | 写入内容 |
|------|---------|
| `SESSION.md` | 更新章节状态为"已润色"、更新评分记录 |
| `.learnings/ERRORS.md` | 质量问题记录（评分低于出稿门槛时） |

> 决策点：DP12-1 润色方向确认 → DP12-2 评分结果处理 → DP12-3 章节完成与下一步。详见 `references/workflow-interaction.md`。

### 卷结束检查点

每完成一卷（或约 50 章）时执行以下汇总操作：

| 操作 | 说明 |
|------|------|
| 更新文风圣经 | 根据本卷实际写作经验微调 `memory/project_style.md`（擦边等级、叙事节奏、描写密度等） |
| 归档记忆文件 | 将 `.learnings/` 中 30+ 章未出场的角色、地点、支线移入归档区 |
| 生成本卷摘要 | 写入 `追踪/卷摘要_第X卷.md`（核心事件、角色弧光、未解伏笔、下卷铺垫） |
| 伏笔盘点 | 检查 `追踪/伏笔.md`，标记本卷已回收伏笔，列出跨卷待回收伏笔 |
| 节奏复盘 | 统计本卷爽点密度、战斗/日常比例，为下卷节奏设计提供参考 |

## 修改流程

### 大纲修改

1. 修改 `大纲/大纲.md` 和受影响的 `大纲/卷纲_第X卷.md` 2. 评估影响：仅影响未写章节→标记"待返修"；影响已写正文→标记"待重写" 3. 受影响批次细纲解冻→重新走 Phase 8→9 4. 如有"待重写"章节→重新走 Phase 10-12 5. 同步写入记忆

| 级别 | 判断标准 | 处理方式 |
|------|---------|---------|
| 局部微调 | 只改细节，不影响后续走向 | 修改大纲，同步更新细纲 |
| 结构调整 | 改变卷级走向或角色命运 | 修改大纲+卷纲，受影响批次解冻返修 |
| 主线重构 | 核心冲突/金手指/结局变化 | 全部未冻结细纲解冻，从头返修 |

### 细纲修改

1. 目标批次细纲解冻→修改→重新走 Phase 9 审核冻结 2. 已写正文受影响→标记"待重写"→重走 Phase 10-12 3. 同步写入记忆

### 正文改稿（章节改进流水线）

对已写正文做系统性改进时使用。适用于：插入新场景、修复设定矛盾、补充铺垫链、章节拆分/合并等。每轮改进完整执行以下五步。

#### Step 1：审核定位

**必读文件**（并行 Read）：目标章节正文、前后 1-2 章正文、相关角色设定、`设定/不可改动清单.md`（**必讀**）、`追踪/CHAPTERS.md`、`追踪/伏笔.md`、`.learnings/CHARACTERS.md`、`SESSION.md`。

**Grep 影响范围扫描**：用多组关键词（正式名+别名+场景词+动作词，至少 3 组）搜索所有正文文件，确定级联影响范围。

**冻结章节检查**：确认改动是否触碰 Frozen_Canon 锁定内容。若触碰 → 停止并告知用户，除非用户明确授权解冻。

**输出审核报告**：目标章节、问题定位、影响范围（哪些章节需联动修正）、冻结区冲突、预计改动量。

#### Step 2：设计修复方案

| 级别 | 判断标准 | 处理方式 |
|------|---------|---------|
| 微调 | 措辞/细节不一致 | 直接修，不设计 |
| 局部改动 | 单章内新增/修改场景 | 设计场景方案 |
| 跨章联动 | 影响多个章节 | 设计完整方案+级联清单 |
| 结构性修改 | 章节拆分/合并/重排 | 设计拆分/合并方案+全量级联 |

局部改动及以上，用 AskUserQuestion 与用户确认方案方向（插入位置、涉及角色、是否留尾巴等）。

#### Step 3：重写正文

**工具选择**（关键规则）：

| 场景 | 工具 | 原因 |
|------|------|------|
| 短文本微调（<100字） | Edit | 简单替换即可 |
| 中文长文本改动（≥100字） | **Read + Write** | Edit 对中文长文本的 `old_string` 匹配不稳定，编码问题导致频繁失败 |
| 新增整段/整场景 | **Read + Write** | 同上 |

Write 操作流程：Read 完整内容 → 内存中修改 → Write 整体覆写 → 再次 Read 验证。

**冻结章节保护**：冻结章节正文不可触碰，仅在用户明确授权时修改。

**改动质量**：新增内容须与前后文风格一致；每段承载至少 2 个功能；遵循去AI味规范。

**跨章节联动修正**：改动目标章后，逐一 Read → 定位矛盾 → Edit/Write 修正后续受影响章节。

**章节拆分/合并/重命名**：重命名从最高编号往下（避免冲突）；用 Python 脚本执行（Python 对 UTF-8 中文路径比 PowerShell 更稳定）；Glob 验证结果。

#### Step 4：同步下游文件

按顺序更新 7 类文件：

| 序号 | 文件 | 更新内容 |
|------|------|---------|
| 1 | `设定/不可改动清单.md` | 新增 Frozen_Canon 条目；如有新写作门控，新增门控子区 |
| 2 | `大纲/大纲.md` | 更新受影响章节行描述；更新长线钩子表 |
| 3 | `大纲/细纲.md` | 更新受影响章节行描述 |
| 4 | `大纲/批次细纲/第XXX-XXX章_v*.md` | 更新受影响章节的节描述 |
| 5 | `追踪/CHAPTERS.md` | 扩展受影响章节的核心剧情链 |
| 6 | `追踪/伏笔.md` | 新增/更新伏笔条目及关联 |
| 7 | `SESSION.md` | 新增改进轮次记录；更新状态 |

**更新原则**：措辞一致性（同一事件在不同文件中描述须一致）；Frozen_Canon/FB 编号续接不跳号；措辞修正时 Grep 全量搜索旧表述并逐文件统一。

> 详细更新规则和格式模板见 `references/downstream-sync-checklist.md`。

#### Step 5：Agent 终审验证

用 Agent 工具启动**独立子任务**做验证（子任务不知主任务修改过程，避免确认偏误）。

**标准 10 项检查**：
1. 目标章节正文是否包含预期改动
2. 联动修正章节是否已更新
3. 不可改动清单是否包含新 Frozen_Canon 条目
4. 伏笔表是否已更新
5. CHAPTERS.md 受影响章节行是否已扩展
6. 细纲.md 受影响章节行是否已更新
7. 大纲.md 受影响章节行是否已同步
8. 批次细纲受影响章节节是否已补充
9. SESSION.md 是否新增改进记录
10. 后续章节正文是否存在矛盾残留（Grep 多组关键词逐章扫描）

**预防性额外检查**：英文残留扫描、旧描述清理确认、FC/FB 编号连续性、冻结章节完整性。

**FAIL 处理**：SubAgent 报告 FAIL → 主任务逐项修复 → 再次 SubAgent 复验。同一问题连续 2 次 FAIL → 暂停报告用户介入。

> 详细验证检查项、Grep 策略和报告模板见 `references/verification-checklist.md`。

### 修改后记忆同步

> 详细规范见 `references/artifact-protocols.md`。须同步：`.learnings/STORY_BIBLE.md`（世界观变更）、`.learnings/CHARACTERS.md`（角色变更）、`.learnings/PLOT_SUSPENSE.md`（情节+伏笔变更）、`大纲/细纲迭代记录.md`（返修记录）、`SESSION.md`（进度更新）
