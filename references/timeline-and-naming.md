# 时间线与命名规范

> 何时加载：Phase 7 细纲规划、Phase 8 生成细纲、卷结束检查点

时间线和章节命名是长篇网文最容易翻车、也最容易被忽视的两个细节。时间线乱了读者会出戏算日子，命名垮了章节没人点。本指南给出时间线条目格式、跳跃提示、章节命名套路、卷首卷尾模板和字数控制五块实操规范。

---

## 一、时间线条目格式

`.learnings/时间线.md`（或 `追踪/时间线.md`）是全书的"日历真相源"。每条时间线必须有统一格式，否则跨章核对时会崩。

### 三种时间锚点

| 锚点类型 | 说明 | 适用场景 | 格式模板 |
|----------|------|----------|----------|
| **绝对时间** | 世界观内历法的具体日期/年份 | 重大事件、卷级时间标记 | `天元历 3024 年·秋·七月十五 — 事件描述` |
| **相对时间** | 以某事件为 0 点的偏移量 | 紧凑剧情、连续事件 | `主角拜师后第 3 日 — 事件描述` |
| **事件锚点** | 用另一个事件做参照 | 平行线索、无法确定具体时间 | `魔教攻山同期 — 暗线：朝廷调兵` |

### 条目格式模板

每条时间线建议包含五要素：锚点、地点、人物、事件、章节出处。

```
[锚点] · [地点] · [人物] — [事件]（第 XX 章）
```

示例：
```
天元历 3024 年·秋·七月十五 · 青云山·主峰 · 林川、苏婉 — 拜师大典，林川正式入门（第 12 章）
拜师后第 3 日 · 藏经阁 · 林川 — 偶得残卷《无字心经》（第 13 章）
魔教攻山同期 · 京城·兵部 · 朝廷暗探 — 秘密调拨三千禁军北上（第 18 章，暗线）
```

### 平行线索标注

当多条线索同时进行，用"同期/前夜/后一日"做相对标注，并列记录：

```
【魔教攻山日】
  - 青云山·主峰 · 林川等 — 防守战（第 18 章）
  - 京城·兵部 · 朝廷 — 调兵北上（第 18 章暗线）
  - 南疆·苗寨 · 苗女阿萝 — 感应到上古封印松动（第 18 章暗线）
```

### 实操要点

1. 每章生成后，把本章发生的事补进时间线，标注章节号。
2. 跨章事件（如一场大战写了 3 章）用同一锚点，注明"第 18-20 章"。
3. 模糊时间（"数日后""半月之后"）要在条目里换算成相对锚点，方便后续核对。
4. 卷首/卷尾用绝对时间做分隔标记，便于读者感知时间跨度。

### 反面案例

> ❌ 时间线只写"林川修炼了一段时间"，没锚点。写到第 50 章发现"林川拜师才两个月却已经天下无敌"，节奏崩坏。

### 正面案例

> ✅ 每章一条时间线条目，绝对时间+相对时间双标。写到第 50 章时一眼看出"拜师第 78 日"，知道该安排一次"三月小成"的时间跳跃来收束节奏。

## 二、时间跳跃读者提示规范

网文时间跳跃频繁（修炼、赶路、养伤、闭关）。跳跃没提示读者会懵，提示太硬又像流水账。

### 跳跃类型

| 跳跃类型 | 跨度 | 用途 | 提示强度 |
|----------|------|------|----------|
| **微跳** | 几小时-几天 | 略过无聊过渡 | 一句话带过即可 |
| **小跳** | 十几天-几个月 | 修炼阶段、养伤、赶路 | 一段过渡 + 关键变化点 |
| **中跳** | 半年-两三年 | 闭关突破、长途游历 | 单独过渡段 + 时间锚点 + 期间大事 |
| **大跳** | 五年-几十年 | 转场换地图、时代更迭 | 章节级过渡 + 卷间过渡 + 重要人物变动 |

### 提示技巧

1. **锚点开场**：跳跃后第一句给时间锚点。"三年后。青云山已是深秋。"
2. **变化对照**：用一两个具体变化让读者感知时间流逝。"当年及膝的师弟，如今已比林川高出半头。"
3. **传闻带过**：用旁人闲话交代跳跃期间发生的大事。"听说了吗？北境三年前那场大战，魔教退了。"
4. **物品/伤口标记**：用一个具体的物或伤疤做时间刻度。"腰间那道疤，是三年前留下的。"

### 禁忌

- ❌ **跳跃后不交代时间**：读者以为还是同一天，结果角色突然说"五年不见"。
- ❌ **跳跃期间埋了伏笔却不标注**：后期回收时读者完全不记得，因为跳跃段被略读了。
- ❌ **大跳后人物性格突变无铺垫**：跳跃 = 跳过铺垫，但跳跃后必须用闪回/回忆补一点过渡，否则突兀。
- ❌ **频繁小跳**：一章里跳 3 次以上，读者像在看快进，无沉浸感。

### 实操要点

- 中跳以上必须更新 `时间线.md`，标注"【时间跳跃】第 XX 章 → 第 XX+1 章，跨度 X 年"。
- 大跳后第一个场景建议安排"故人重逢"或"旧地重游"，用熟悉元素帮读者重新建立坐标。
- 跳跃期间发生的、对后续有影响的事件，要在跳跃后用一两句话交代，不能完全黑箱。

## 三、章节命名套路

章名是读者点击的第一诱饵。好章名能拉高点击率 30% 以上，烂章名（"第 18 章"）等于自杀。

### 四种命名风格

| 风格 | 特点 | 适用 | 示例 |
|------|------|------|------|
| **悬念式** | 抛出一个让读者好奇的结果/疑问 | 高潮章、转折章 | 《谁杀了掌门？》《真相》《不可信任的人》 |
| **场景式** | 点出本章核心场景/地点 | 过渡章、铺垫章 | 《藏经阁》《血战青云顶》《京城第一夜》 |
| **情绪式** | 用情绪词定调 | 情感高潮章 | 《诀别》《重逢》《我不甘心》 |
| **反差式** | 标题与内容形成反差/双关 | 反转章 | 《喜宴》（实为鸿门宴）、《恩人》（实为仇人） |

### 平台偏好

| 平台 | 偏好风格 | 备注 |
|------|----------|------|
| **起点** | 悬念式 + 场景式为主，2-6 字短章名流行，爽点直给 | 读者看重信息量，章名可适度剧透爽点 |
| **番茄** | 情绪式 + 悬念式，章名可稍长，强调冲突和反转 | 免费阅读靠章名留人，每章名都是钩子 |
| **晋江** | 情绪式 + 反差式，文艺感、留白感，常引用诗句/歌词 | 女频读者看重氛围，章名忌太直白剧透 |
| **纵横/17K** | 场景式 + 悬念式，介于起点和番茄之间 | 中规中矩，重信息量 |

### 实操要点

1. 章名长度：2-8 字最稳。太短（1 字）显装，太长（10+ 字）显啰嗦。
2. 避免"第 XX 章"裸号——至少加个章名。裸号章在书架模式毫无辨识度。
3. 高潮章用悬念式，过渡章用场景式，按章性质选风格，不要全书一个调。
4. 章名不剧透关键反转：可以暗示，但不要把底牌写在标题里（如《凶手是师弟》就过了）。
5. 系列章（连战 3 章）可用递进章名：《初战》《再战》《终战》或《围》《困》《破》。

### 反面案例

> ❌ 全书 200 章都是"第 X 章"裸号。读者从书架找某场戏，得翻半天。

### 正面案例

> ✅ 高潮章《谁杀了掌门？》→ 过渡章《藏经阁》→ 情感章《诀别》→ 反转章《喜宴》。风格随章节性质切换，每章都有辨识度。

## 四、卷首卷尾设计模板

卷是长篇的呼吸单位。卷首帮读者重新定位，卷尾给读者一个收束感和追读下一卷的理由。

### 卷首回顾（卷开头 1-2 段）

四要素：
1. **时间锚点**：本卷开始的世界时间。
2. **主角状态**：主角现在什么境界/身份/处境。
3. **上卷遗留**：上卷结尾留的悬念/未解之事。
4. **本卷目标**：本卷主角要解决的核心问题。

模板：
```
【第 X 卷·卷名】

距上卷结尾已过 X 时。林川拜入青云宗三载，修至筑基中期，已是内门首席弟子。
而三年前那场魔教攻山的真相，至今未明。师父留下的残卷《无字心经》，也只参透了前三页。
这一卷，他要南下苗疆，寻找残卷第四页的下落。
```

### 卷末收束（卷结尾 1-2 段）

三要素：
1. **本卷闭环**：本卷核心问题解决到什么程度。
2. **代价结算**：主角为此付出了什么、失去了什么。
3. **下卷钩子**：抛出一个新的、更大的悬念。

模板：
```
【第 X 卷·终章】

苗疆之行，林川拿到了残卷第四页，却失去了同行的师弟阿萝。
她坠入封印深渊的那一刻，林川才明白——残卷要的不是"参透"，是"献祭"。
而封印深处传来的那道声音，自称是他从未谋面的父亲。
（第 X 卷·完，下一卷：深渊之约）
```

### 卷间过渡

- **直接切场**：上卷末留钩子，下卷首直接进入新场景，节奏快。
- **时间跳跃过渡**：卷间跳跃数月数年，卷首用"已过 X 时"交代。
- **视角切换**：上卷主角视角，下卷首切到反派/配角视角做铺垫，再切回主角。

### 实操要点

1. 每卷必须有独立的小高潮和闭环，不能"全卷都在铺垫"。
2. 卷末钩子要比章末钩子强一档——读者要等更久才能看到下卷。
3. 卷名要能概括本卷主题（如《苗疆风云》《深渊之约》），不要用"第 X 卷"裸号。
4. 卷首回顾不要太长，2 段以内，老读者会跳过，新读者要能快速进入。

## 五、章节字数控制

字数不是越多越好。不同字数档位对应不同节奏，选错档位会让一章显得拖沓或仓促。

### 三档字数节奏

| 档位 | 字数 | 节奏特征 | 适用场景 |
|------|------|----------|----------|
| **2000 字** | 1800-2200 | 紧凑、单线、快推 | 高潮章、动作章、爽点章 |
| **3000 字** | 2800-3200 | 标准、可双线、信息适中 | 常规章、铺垫章、对话章 |
| **4000 字** | 3800-4500 | 厚重、多线、信息密集 | 转折章、卷首卷尾、世界观揭示章 |

### 档位选择原则

- **爽点章用 2000 字**：爽点要快，拖长就泄气。一击制胜、装逼打脸、突破升级，越短越爽。
- **日常章用 3000 字**：给角色互动、伏笔铺垫留空间，但不至于注水。
- **关键章用 4000 字**：信息量大、多线交汇时才用，否则读者消化不了。
- **不要全书一个档**：连续 5 章 4000 字会让人觉得水，连续 5 章 2000 字会让人觉得浅。

### 平台偏好

| 平台 | 单章主流字数 | 备注 |
|------|--------------|------|
| **起点** | 3000-4000 字 | 付费章节，读者期待信息量 |
| **番茄** | 2000-3000 字 | 免费阅读，快节奏留人，章末强钩子 |
| **晋江** | 3000-5000 字 | 女频读者接受较长章节，重氛围 |
| **纵横/17K** | 3000-4000 字 | 与起点接近 |

### 实操要点

1. 写细纲时就标注每章预期字数档位，避免写嗨了收不住。
2. 一章超过 4500 字考虑拆两章——拆出的新章能多一个章末钩子。
3. 一章不足 1500 字不要单独发，合并到相邻章。
4. 卷首卷尾可以比常规章长一档，给读者"进入/离开"的仪式感。

### 反面案例

> ❌ 全书每章 4000 字，连打斗爽点章都拉到 4000 字，结果爽点被环境描写和内心戏稀释，读者觉得"水"。

### 正面案例

> ✅ 日常章 3000 字铺垫，爽点章压缩到 2000 字快打快收，卷终转折章放宽到 4000 字多线收束。节奏张弛有度。

## 实操清单（细纲/卷节点过一遍）

- [ ] 本章时间线条目是否已按五要素格式补入 `时间线.md`？
- [ ] 本章是否有时间跳跃？跳跃类型和提示强度是否匹配？
- [ ] 章名是否选对了风格？是否避免裸号和过度剧透？
- [ ] 卷首是否包含时间锚点 + 主角状态 + 上卷遗留 + 本卷目标？
- [ ] 卷末是否有闭环 + 代价 + 下卷钩子？
- [ ] 本卷卷名是否有主题感？
- [ ] 本章字数档位是否与章节性质匹配（爽点短、日常中、关键长）？
- [ ] 平台偏好是否对齐目标平台主流字数？
