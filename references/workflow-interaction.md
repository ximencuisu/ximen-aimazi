# 全流程交互决策点指南

> 何时加载：创作流程启动时，与 `SKILL.md` 配合使用。续写流程另见 `continuation-interaction.md`。

## 概述

创作流程中的每个关键步骤都应设置交互决策点，让用户能全程参与讨论和掌控创作方向。本文档定义了 Phase 1-12 共 23 个决策点（含 4 个读取确认硬门控 DP6-0/7-0/8-0/10-1）的交互模板、选项设计和衔接话术。

---

## 交互模式

### 两种模式

| 模式 | 特点 | 适合用户 |
|------|------|---------|
| 🗨️ 讨论模式（默认） | 每个决策点都展示选项+讨论 | 想深度参与方向的作者 |
| ⚡ 快速模式 | 只在关键决策点暂停，其余自动推进 | 想快速出稿的作者 |

**切换方式**：用户可在任意时刻说"切换到快速模式"/"切换到讨论模式"

**快速模式下的必停点**（不可跳过）：
- DP6-0 大纲读取确认（硬门控，必须输出读取确认块）
- DP7-0 细纲规划读取确认（硬门控，必须输出读取确认块）
- DP8-0 生成细纲读取确认（硬门控，必须输出读取确认块）
- DP10-1 正文读取确认（硬门控，必须输出读取确认块）
- DP1-2 创意方案选择
- DP4-1 创作参数配置
- DP9-1 审核结果与方案选择
- DP10-2 章节生成后审核（首章）

### 交互格式规范

每个决策点的交互必须包含四个要素：

1. **当前状态摘要**：用 2-3 句话说明"我们现在在哪、在做什么"
2. **选项列表**：2-4 个选项 + 自定义入口（使用当前 Agent 平台的结构化提问能力呈现；没有专用工具时用清晰编号列表）
3. **推荐标注**：AI 推荐的选项在描述末尾标注"（推荐）"
4. **上下文提示**：为什么给出这些选项

## 23 个决策点

### DP1-1：创作方向确认

**时机**：用户提供初始方向后，AI 补全 8 维度提示词后
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"收到你的方向！让我确认一下我的理解，并补充一些你可能没想到的维度。"

**交互内容**：

```
🎯 创作方向确认：

你的方向：{用户原始输入}
我的理解：{AI理解摘要}

8维度补全：
- 题材定位：{补全}
- 世界观方向：{补全}
- 主角人设方向：{补全}
- 核心冲突方向：{补全}
- 爽点设计方向：{补全}
- 节奏规划方向：{补全}
- 配角框架方向：{补全}
- 开篇钩子方向：{补全}

❓ 我的理解和补充方向对吗？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 方向正确，继续生成方案 | 进入 DP1-2 |
| B. 某些维度需要调整 | 用户指出后逐项调整 |
| C. 方向不太对，让我重新描述 | 重新收集方向 |
| D. 先给我看看参考题材 | 加载 genre-frameworks-unified.md 展示 |

### DP1-2：创意方案选择

**时机**：3 套创意方案生成后
**模式**：所有模式必停

**状态摘要**：
"基于确认的方向，我准备了 3 套创意方案，每套有不同的侧重点。"

**交互内容**：

```
💡 3套创意方案：

【方案一】《{书名}》
  一句话简介：{简介}
  核心爽点：{爽点}
  适合读者：{读者画像}
  风险提示：{风险}

【方案二】《{书名}》
  一句话简介：{简介}
  核心爽点：{爽点}
  适合读者：{读者画像}
  风险提示：{风险}

【方案三】《{书名}》
  一句话简介：{简介}
  核心爽点：{爽点}
  适合读者：{读者画像}
  风险提示：{风险}

❓ 你倾向哪套？也可以混合。
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 方案一 | 选择方案一 |
| B. 方案二 | 选择方案二 |
| C. 方案三 | 选择方案三 |
| D. 混合/自定义 | 用户说明怎么混 |

如果选择混合，追问："你想怎么混？比如：用哪套的核心设定？用哪套的爽点设计？"
如果都不满意，最多迭代 3 轮重新生成方案。

### DP4-1：创作参数配置

**时机**：进入 Phase 4
**模式**：所有模式必停

**状态摘要**：
"在生成大纲前，需要确认几个关键创作参数，这些会直接影响后续所有内容。"

**交互内容**：

```
⚙️ 创作参数配置：

参数1：章节数量
  A. 50章（短篇/试水）
  B. 100章（标准）
  C. 200章（长篇）
  D. 500章（超长篇）

参数2：女角色数量
  A. 1个（唯一女主）
  B. 2个（双女主）
  C. 3个（经典三女主）
  D. 5个（后宫向）

参数3：擦边等级
  A. 无（纯剧情）
  B. 少量（5-10%）
  C. 中量（15-25%）
  D. 大量（30-40%）

参数4：写作风格
  A. 热血爽文风
  B. 都市现实风
  C. 古风仙侠风
  D. 悬疑惊悚风
  E. 轻松日常风
  F. 对比学习文风（贴入 AI 原文 + 用户修改文，自动提取规则）

参数5：作家技法参考
  A. AI推荐 1 主 + 1 辅
  B. 用户自选 1 主 + 1 辅
  C. 不启用作家参考
```

如选择擦边（非"无"）→ 追问擦边描写风格：A 含蓄 / B 直白 / C 文艺 / D 暴力美学 / E 自定义

如选择文风 F → 读取 `assets/STYLE-CONTRAST-TEMPLATE.md` 与 `references/style-contrast-analysis.md`；用户未提供对比样本时，启用 `memory/style_contrast_guide.md` 的预置基线规则。

**选项**：使用当前 Agent 平台的结构化提问能力一次性询问 5 个参数；没有专用工具时用清晰编号列表。启用作家参考后，必须写入 `memory/project_style.md`。

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 用户在决策点选择"自定义" | 收集用户描述后调整方案 |
| 用户连续 3 次选择"不满意" | 建议回退到上一个决策点重新讨论 |
| 用户在快速模式下要求讨论 | 立即切换到讨论模式 |
| 用户跳过决策点 | 记录到 SESSION.md，使用 AI 推荐选项 |
| 用户在决策点提出新需求 | 评估是否需要回退到更早的阶段 |
| 用户从中间切入 | 执行进度检测，从恢复位置开始，走完剩余流程 |
| 用户明确表示跳过某 Phase | 记录到 SESSION.md，跳过该 Phase 所有决策点 |
| 用户说"你来定"/"按推荐来" | 使用 AI 推荐选项，记录到 SESSION.md |
