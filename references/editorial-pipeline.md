# AI 编辑部流水线

把单个写作 Agent 按编辑部角色顺序运行，用来做更稳定的立项、出稿、审稿和返修。

注意：这里是“角色化工作流”，不是要求真的开多个子代理。默认由同一个 Agent 顺序扮演不同岗位。

---

## 适用场景

- 你不想直接“开写”，而是先像工作室一样走流程
- 你已经有题材，但想让大纲和审稿更职业一点
- 你要批量产出章节，希望每章都有固定质检环节
- 你准备做长篇连载，需要稳定的生产节奏

---

## 五个核心岗位

| 角色 | 职责 | 典型输入 | 典型输出 |
|------|------|------|------|
| 题材参谋 | 评估方向、卖点、平台感 | 点子、竞品、题材偏好 | `output/立项单.md` |
| 架构编辑 | 定世界观、人物关系、分卷大纲 | 立项单、风格目标 | `output/大纲.md`、`output/细纲.md` |
| 章节执行 | 按细纲写正文 | 审核通过的细纲、文风配置 | `output/第XX章_[章名].md` |
| 毒舌审稿 | 找问题、标风险、抓水文 | 草稿、真相档案 | `output/审稿报告.md` |
| 改稿编辑 | 根据报告修订并解释修改点 | 审稿报告、原稿 | `output/返修记录.md` |

---

## 流水线怎么跑

### 第一阶段：立项

由题材参谋负责：

- 确认题材、目标读者、核心卖点
- 判断这本书的主要承诺是什么
- 明确前 20 章最该让读者看到什么

如果用户提供竞品、榜单、风格样本，可以利用；如果没有，就基于现有风格库和题材经验给出判断。

### 第二阶段：搭框架

由架构编辑负责：

- 先建世界观和主要矛盾
- 再拆分卷目标
- 再落到章节细纲

这一步必须与 `.learnings/` 和 `SESSION.md` 兼容，避免大纲系统与真相档案脱节。

### 第三阶段：出章节

由章节执行负责：

- 严格按当前细纲写
- 写前读取激活文风和作者技法参考
- 写后同步真相档案

### 第四阶段：审稿

由毒舌审稿负责：

- 先跑基础质量评分
- 再按 `references/advanced-audit.md` 做深度审稿
- 明确标出严重、中等、轻微问题

### 第五阶段：返修

由改稿编辑负责：

- 优先修严重问题
- 再修影响阅读节奏的问题
- 最后处理文风和措辞

返修后再次检查章末钩子、爽点兑现和前后文一致性。

## 推荐的出稿门槛

| 场景 | 最低门槛 |
|------|------|
| 普通日更 | `quality-check.md` 总分 >= 7.0 |
| 关键高潮章 | 总分 >= 8.0，且深度审稿无红项 |
| 卷终章 | 总分 >= 8.2，并检查伏笔回收 |
| 发布前精修 | 必跑 `advanced-audit.md` |

## 角色提示模板

### 题材参谋

```text
从目标读者、核心卖点、题材竞争感三个角度，帮我判断这个点子值不值得写。
```

### 架构编辑

```text
按长篇连载思路，把这个点子拆成分卷目标、关键转折和前30章节奏。
```

### 章节执行

```text
按当前细纲写正文，保留文风和爽点，不要跳出已建立设定。
```

### 毒舌审稿

```text
不要夸，直接按严重/中等/轻微问题审这一章，优先抓逻辑、节奏、AI味和毒点。
```

### 改稿编辑

```text
根据审稿报告修稿，优先解决会影响追读的硬问题，再保留原有亮点。
```
