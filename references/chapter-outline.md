# 章节细纲生成与逻辑审核

大纲确认后，生成每章详细细纲，并进行逻辑审核，确保无矛盾后再写正文。

---

## 细纲与逻辑审核流程

```
大纲确认 → 生成细纲 → 逻辑审核 → 修改细纲 → 确认细纲 → 写正文
                                                        ↑
                                                   审核不通过
                                                      ↓
                                               返回修改细纲
```

---

## 第一部分：细纲生成

### 细纲定义

细纲是章节的"骨架"，包含：
- 本章核心事件
- 关键情节点
- 人物出场和状态变化
- 地点转移
- 伏笔埋设/回收
- 爽点设计

### 细纲模板

```markdown
## 第XX章 [章节名]

### 基本信息
- **章节类型**：剧情推进 / 战斗 / 情感 / 转折 / 过渡
- **核心事件**：本章发生的最重要的一件事
- **爽点类型**：打脸 / 突破 / 逆转 / 获宝 / 情感升温
- **情绪曲线**：低开高走 / 层层递进 / 反转爆发 / 紧张释放

### 剧情要素
- **开场**：如何开始（上章结尾 / 新场景 / 对话）
- **发展**：情节如何推进（3-5个情节点）
- **高潮**：本章最大的爽点/转折
- **收束**：如何结束，章末钩子

### 人物动态
- **出场人物**：[角色列表]
- **角色变化**：
  - [角色A]：等级/状态变化
  - [角色B]：关系/态度变化
- **新角色**：如有，简要设定

### 世界动态
- **涉及地点**：[地点列表]
- **地点变化**：如有新建/破坏/解锁

### 伏笔管理
- **埋设伏笔**：[列出本章埋下的伏笔]
- **回收伏笔**：[列出本章回收的伏笔]

### 战力/资源变化
- **获得**：[物品/技能/资源]
- **消耗**：[如有消耗]
- **等级变化**：[如有]

### 疑点检查（自检）
- [ ] 情节是否与上章衔接？
- [ ] 人物状态是否与记忆文件一致？
- [ ] 是否有逻辑漏洞？
- [ ] 伏笔是否有对应回收？
```

## 第二部分：逻辑审核

### 审核维度

#### 1. 时间线审核

| 检查项 | 问题示例 | 正确示例 |
|--------|---------|---------|
| 章节衔接 | 上章说"三天后"，下章变"次日" | 保持时间线一致 |
| 成长时间 | 修炼一年vs突破只需三天 | 符合力量体系设定 |
| 行动时间 | A在城东，B在城西，同一时间相遇 | 空间距离合理 |

#### 2. 空间线审核

| 检查项 | 问题示例 | 正确示例 |
|--------|---------|---------|
| 地点连续 | 上章在宗门，下章突然在皇宫 | 地点转移有交代 |
| 距离合理 | 刚在城东，下一句在城西 | 行动需要时间 |
| 空间设定 | 宗门在山上，却说"下船" | 符合世界观 |

#### 3. 人物线审核

| 检查项 | 问题示例 | 正确示例 |
|--------|---------|---------|
| 存活状态 | 已死角色出现在本章 | 确认角色状态 |
| 等级匹配 | 炼气期打败化神期，无合理解释 | 战力合理 |
| 关系逻辑 | 刚结仇，下章称兄道弟 | 关系变化有铺垫 |
| 能力一致 | 主角不会炼丹，突然炼出丹药 | 能力有来源 |

#### 4. 设定线审核

| 检查项 | 问题示例 | 正确示例 |
|--------|---------|---------|
| 力量体系 | 筑基期用筑基期的能力 | 符合等级设定 |
| 社会规则 | 弱者当众打脸强者，无后果 | 符合社会规则 |
| 物品设定 | 凭空出现未交代来源的物品 | 物品有来源 |
| 系统规则 | 系统能力超出设定范围 | 遵守系统规则 |

#### 5. 伏笔线审核

| 检查项 | 问题示例 | 正确示例 |
|--------|---------|---------|
| 伏笔呼应 | 埋下伏笔后从未回收 | 伏笔有回收计划 |
| 回收时机 | 刚埋下就回收，无悬念 | 伏笔有足够铺垫 |
| 伏笔冲突 | 回收的伏笔与后续设定矛盾 | 伏笔逻辑自洽 |

## 第三部分：审核工具

### 逻辑矛盾自检表

生成细纲后，逐章检查：

```markdown
## 逻辑自检

### 章节：第XX章
- [ ] 时间线一致（上章→本章→下章）
- [ ] 空间线一致（地点转移合理）
- [ ] 人物存活（无已死角色出场）
- [ ] 等级匹配（战力无崩坏）
- [ ] 关系合理（关系变化有铺垫）
- [ ] 能力来源（能力有明确来源）
- [ ] 物品来源（获得物品有交代）
- [ ] 设定遵守（遵守世界规则）
- [ ] 伏笔自洽（伏笔逻辑不矛盾）
- [ ] 爽点合理（爽点有铺垫，不突兀）

### 跨章节检查
- [ ] 与前3章无矛盾
- [ ] 与大纲设定无冲突
- [ ] 伏笔埋设和回收对应
```

### 常见逻辑矛盾类型

| 类型 | 描述 | 检查方法 |
|------|------|---------|
| 时间矛盾 | 同一时间发生两件不可能同时发生的事 | 检查时间标注 |
| 空间矛盾 | 角色同时出现在两个不同地点 | 检查地点标注 |
| 状态矛盾 | 角色状态前后不一致 | 检查角色状态表 |
| 因果矛盾 | 结果与原因不符 | 检查因果链 |
| 设定矛盾 | 违背已建立的规则 | 检查设定表 |
| 伏笔矛盾 | 伏笔回收与后续冲突 | 检查伏笔链 |

## 第四部分：细纲生成规范

### 生成原则

1. **先整体后局部** - 先生成所有细纲，再逐章审核
2. **保持连贯性** - 每章细纲要与前后章节衔接
3. **埋设伏笔** - 在前期章节埋设伏笔，中后期回收
4. **控制节奏** - 避免连续多章平淡或连续多章高潮

### 细纲数量

| 总字数目标 | 章节数 | 每卷章数建议 |
|-----------|--------|-------------|
| 30万字 | 100章 | 10章/卷，共10卷 |
| 50万字 | 166章 | 10章/卷，共17卷 |
| 100万字 | 333章 | 10章/卷，共34卷 |

### 每卷细纲结构

```markdown
# 第一卷：[卷名]

## 卷概述
- **核心冲突**：本卷要解决的主要矛盾
- **成长目标**：主角在本卷的成长
- **卷末状态**：本卷结束时的情况

## 章节细纲

### 第1-3章：开篇
### 第4-6章：发展
### 第7-9章：高潮
### 第10章：卷末决战
```

## 第六部分：审核流程示例

### 场景：玄幻修仙小说

```
用户：大纲已确认，生成细纲

AI：
1. 生成全本333章细纲
2. 逐章检查逻辑
3. 发现问题：
   - 第45章：主角"穿越到10年前"，但第20章提到"10年前父亲已死"
   - 第78章：说赵天罡"首次出场"，但第15章已有赵天罡
4. 修改第45章为"穿越到父亲死前1个月"
5. 删除第78章的"首次出场"描述
6. 确认细纲无误
7. 开始写正文
```

## 第七部分：常见问题处理

### Q1：发现与大纲冲突怎么办？

**处理**：
1. 检查是否理解错误
2. 如细纲有误，修改细纲
3. 如大纲有误，修改大纲
4. 记录修改原因

### Q2：发现伏笔无法回收怎么办？

**处理**：
1. 检查伏笔是否必要
2. 如不必要，删除该伏笔
3. 如必要，修改伏笔设计
4. 在后续章节安排回收

### Q3：发现战力崩坏怎么办？

**处理**：
1. 检查等级设定
2. 确认是否有特殊buff
3. 如无合理解释，修改情节
4. 记录战力规则

### Q4：多章连续平淡怎么办？

**处理**：
1. 在其中插入小高潮
2. 增加打脸场景
3. 加快节奏
4. 重新调整爽点分布
