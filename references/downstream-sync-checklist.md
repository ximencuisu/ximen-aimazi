# 下游文件同步清单

> 何时加载：改稿流水线 Step 4

本文件为 `webnovel-chapter-polish` 技能的 Step 4（同步下游文件）提供详细更新规则和格式模板。

---

## 同步顺序

按以下顺序更新，避免依赖错乱：

1. 不可改动清单（先锁定新确立的事实）
2. 大纲 → 细纲 → 批次细纲（从粗到细）
3. CHAPTERS.md → 伏笔.md（追踪层）
4. SESSION.md（最后更新状态）

---

## 1. 不可改动清单

**文件**：`设定/不可改动清单.md`

### 新增 Frozen_Canon 条目

每次改稿确立了新的不可变事实时，在 `Frozen_Canon` 区追加条目，编号续接已有最大值：

```
FC{编号}：{一句话描述锁定的事实}
  - 来源：第{X}章 改稿第{N}轮
  - 范围：{影响的具体章节/设定}
```

示例：
```
FC029：武馆清洗事件——沈重鸣逼宫失败，带王虎等弟子投青狼帮，武馆仅剩三人
  - 来源：第008章 改稿第4轮
  - 范围：第008章正文、第009章联动、伏笔FB015
```

### 新增写作门控

如果改稿确立了结构性规则（如节奏门控），在门控子区追加：

```
第{起始章}-{结束章}{门控名称}：
  - 规则1：{具体规则}
  - 规则2：{具体规则}
```

### 措辞修正级联

当修正某处措辞时（如"弟子"→"三人"），用 Grep 搜索不可改动清单中是否有旧措辞，有则一并修正。

---

## 2. 大纲

**文件**：`大纲/大纲.md`

### 更新规则

- 找到受影响章节的行，追加或修改描述
- 如改动影响卷级结构，同步更新对应卷纲
- 如新增长线钩子/伏笔线，在长线钩子表中追加

### 格式要求

保持与已有行一致的格式。典型格式：

```
第{XXX}章：{一句话核心剧情描述}
```

如一行内信息过多，可拆为多行：

```
第{XXX}章：{核心剧情}
  - {补充细节1}
  - {补充细节2}
```

---

## 3. 细纲

**文件**：`大纲/细纲.md`

### 更新规则

- 找到受影响章节的行，追加或修改描述
- 描述粒度比大纲更细，包含具体场景和情节转折
- 如改动影响前后章的衔接关系，同步更新相邻章节行

### 措辞一致性

细纲中的措辞必须与大纲保持一致。如大纲写"武馆三人涉险"，细纲不能写"武馆弟子涉险"。

---

## 4. 批次细纲

**文件**：`大纲/批次细纲/第XXX-XXX章_v*.md`

### 更新规则

- 找到受影响章节的节，补充改动描述
- 批次细纲是最细粒度的规划文件，应包含完整的场景链描述
- 如改动涉及爽点/钩子/伏笔的变动，同步更新对应标记

### 格式参考

```
### 第{XXX}章：{章名}
- 定位：{章节功能}
- 爽点：{爽点类型}
- 钩子：{章末钩子类型}
- 场景链：{场景1} → {场景2} → {场景3}
- 伏笔：{本章埋设/回收的伏笔}
- 改动记录：[第{N}轮改稿] {改动描述}
```

---

## 5. CHAPTERS.md

**文件**：`追踪/CHAPTERS.md`

### 更新规则

- 找到受影响章节的行，扩展核心剧情链
- CHAPTERS.md 是章节索引，描述应精炼但完整

### 格式参考

```
| {XXX} | {章名} | {核心剧情一句话} | {关键角色} | {伏笔关联} |
```

如信息较多，可在核心剧情列追加：

```
| 008 | 老裴不能拔刀 | 雨巷伏击+源力首体验+武馆清洗（沈重鸣逼宫→弟子叛逃→仅剩三人） | 左伤尘、老裴、姜照夜、沈重鸣 | FB015 |
```

---

## 6. 伏笔表

**文件**：`追踪/伏笔.md`

### 新增伏笔条目

如改稿新增了伏笔，在伏笔表追加，编号续接：

```
FB{编号}：{伏笔名称}
  - 埋设：第{X}章 {具体描述}
  - 预计回收：第{Y}章 {回收方向}
  - 状态：待回收 / 已回收 / 已废弃
```

### 更新已有伏笔

如改稿影响已有伏笔的埋设/回收状态，更新对应条目。

### 措辞同步

伏笔表中涉及的章节描述须与大纲/细纲/CHAPTERS 保持一致。

---

## 7. SESSION.md

**文件**：根目录 `SESSION.md`

### 更新规则

- 在改进记录区追加本轮改稿摘要
- 更新当前进度状态
- 更新关键基准（如有）

### 格式参考

```
## 改进记录

### 第{N}轮：{改动主题}
- 日期：{日期}
- 涉及章节：第{X}章、第{Y}章
- 改动摘要：{一句话描述}
- 新增 Frozen_Canon：FC{编号}
- 新增伏笔：FB{编号}
- 状态：已完成 / 待复验
```

## 常见陷阱

1. **措辞不一致**：同一事件在不同文件中用了不同表述。修正时须 Grep 全量搜索，逐文件统一。
2. **遗漏批次细纲**：大纲和细纲都更新了，但批次细纲漏了。批次细纲是最细粒度，最容易遗漏。
3. **伏笔编号跳号**：新增伏笔时未续接最大编号，导致后续编号混乱。先读伏笔表确认当前最大编号。
4. **Frozen_Canon 重复**：新增条目与已有条目描述同一事实。新增前先搜索是否已存在。
5. **SESSION 未更新**：改稿完成后忘记更新 SESSION.md。把 SESSION 更新作为 Step 4 的最后一步。
