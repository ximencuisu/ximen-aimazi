# 续写交互决策点指南

> 何时加载：Continuation_Mode 激活时，与 `continuation-engine.md` 配合使用。

## 概述

续写流程中的每个关键步骤都应设置交互决策点，让用户能全程参与讨论和掌控续写方向。本文档定义了 12 个决策点（CP-①~CP-⑫）的交互模板、选项设计和衔接话术。

续写流程复用全流程交互模式（讨论模式/快速模式），决策点编号使用 `CP-` 前缀与主流程的 `DP` 前缀区分。主流程决策点见 `references/workflow-interaction.md`。

---

## 交互模式

### 两种模式

| 模式 | 特点 | 适合用户 |
|------|------|---------|
| 🗨️ 讨论模式（默认） | 每个决策点都展示选项+讨论 | 想深度参与方向的作者 |
| ⚡ 快速模式 | 只在关键决策点暂停，其余自动推进 | 想快速出稿的作者 |

**切换方式**：用户可在任意时刻说"切换到快速模式"/"切换到讨论模式"

**快速模式下的必停点**（不可跳过）：CP-④ 路线选择 / CP-⑥ 不可改动清单确认 / CP-⑩ Sample Chapter 确认 / CP-⑪ 首章关键情节点确认。4 个必停点的理由详见 `continuation-overview.md`"快速模式 4 个必停点"表。

### 交互格式规范

每个决策点的交互必须包含四个要素：

1. **当前状态摘要**：用 2-3 句话说明"我们现在在哪、在做什么"
2. **选项列表**：2-4 个选项 + 自定义入口（使用当前 Agent 平台的结构化提问能力呈现；没有专用工具时用清晰编号列表）
3. **推荐标注**：AI 推荐的选项在描述末尾标注"（推荐）"
4. **上下文提示**：为什么给出这些选项

## 12 个决策点

### CP-①：CP1 提取结果审核

**时机**：CP1 全部产出文件生成完毕后
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"反向解析完成！我从你的原作中提取了作品DNA、风格指纹、角色状态和伏笔信息。让我展示核心发现，请你确认是否准确。"

**交互内容**：

展示反向解析摘要，逐模块呈现：

```
📋 反向解析完成！以下是我从原作中提取的核心信息：

【作品DNA】
- 类型承诺：{提取结果}
- 主角驱动力：{提取结果}
- 主线问题：{提取结果}
- 读者爽点契约：{提取结果}

【风格指纹关键指标】
- 平均句长：{值}字 | 对话占比：{值}%
- 高频意象：{前5个}
- 动作:对话:情绪 = {比例}

【角色当前状态】
- 主角：{当前状态摘要}
- 主要配角：{状态摘要}

【伏笔盘点】
- 待回收伏笔：{数量}条
- 最紧迫的3条：{列表}

❓ 这些理解对吗？有没有我理解错或遗漏的地方？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 基本正确，继续 | 进入决策点②或直接进入CP2 |
| B. 有几处需要修正 | 进入决策点② |
| C. 遗漏了重要信息 | 进入决策点②，侧重补充 |
| D. 重新解析 | 回退CP1，用户提供更多原文 |

---

### CP-②：CP1 关键设定修正

**时机**：用户在决策点①中选择了 B 或 C
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"好的，让我逐项确认你的修正意见。"

**交互内容**：

针对用户指出的修正项，逐项确认：

```
你提到{用户修正内容}。
让我确认修正后的版本：

- {修正项}：{修正后内容}

❓ 这样修正对吗？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 对，就这样 | 写入修正，检查下一项 |
| B. 还需要调整 | 继续讨论 |
| C. 跳过修正，用原版 | 放弃修正 |

修正完成后更新产出文件，进入 CP2。

---

### CP-③：CP2 卡点诊断讨论

**时机**：卡点归类完成后，给路线之前
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"基于反向解析结果，我诊断出了你的卡点类型。让我先确认诊断是否准确，再给你续写路线。"

**交互内容**：

```
🔍 卡点诊断结果：

你的卡点属于【{卡点类型}】{单类型/复合型}：
- {卡点1}：{具体表现}
- {卡点2}：{具体表现}（如有）

根因分析：{根因}

❓ 这个诊断符合你的感受吗？还是你觉得卡在其他地方？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 诊断准确，给路线吧 | 进入决策点④ |
| B. 主要是{某类}卡，其他还好 | 调整诊断权重后进入④ |
| C. 我觉得更像是{其他类型}卡 | 重新诊断 |
| D. 诊断不对，让我描述一下 | 收集用户描述后重新诊断 |

---

### CP-④：CP2 路线选择与调整

**时机**：三条路线展示后
**模式**：所有模式必停

**状态摘要**：
"基于诊断结果，我准备了三条续写路线。每条路线有不同的节奏和风险，请选择最符合你想法的。"

**交互内容**：

```
📌 三条续写路线：

【稳妥线】{路线概述}
  优点：{优点}
  风险：{风险}

【提速线】{路线概述}
  优点：{优点}
  风险：{风险}

【爆点线】{路线概述}
  优点：{优点}
  风险：{风险}

❓ 你倾向哪条？也可以混合，比如"稳妥线节奏+爆点线事件"
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 稳妥线 | 选择稳妥线 |
| B. 提速线 | 选择提速线 |
| C. 爆点线 | 选择爆点线 |
| D. 混合 | 用户说明怎么混 |

如果选择混合，追问："你想怎么混？比如：用哪条线的节奏？用哪条线的核心事件？"

如果都不满意，最多迭代3轮重新生成路线。

---

### CP-⑤：CP2 节拍审核

**时机**：用户选定路线后，展示3-10章节拍
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"路线已选定！下面是未来几章的节拍安排，请审核每章的方向是否合适。"

**交互内容**：

```
📝 未来{N}章节拍（{路线名}）：

第{X}章「{章名}」
  - 核心事件：{事件}
  - 出场角色：{角色}
  - 章末钩子：{钩子}

第{X+1}章「{章名}」
  ...

❓ 整体节拍可以吗？有没有想调整的章节？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 整体可以，继续 | 进入CP3 |
| B. 某几章需要调整 | 用户指出后逐章调整 |
| C. 节奏太快/太慢 | 整体调整节拍密度 |
| D. 想加/删某些章节 | 修改章节数和内容 |

---

### CP-⑥：CP3 锁定理由展示

**时机**：展示不可改动清单时
**模式**：所有模式必停

**状态摘要**：
"在开始续写之前，我们需要确认哪些设定是绝对不能改的。以下是我建议的锁定项，每条都附有理由。"

**交互内容**：

```
🔒 不可改动设定清单：

1. [建议锁定] {条目}
   理由：{为什么不能改}

2. [建议锁定] {条目}
   理由：{为什么不能改}

3. [建议可调整] {条目}
   理由：{为什么可以调整}

❓ 你觉得这些锁定建议合理吗？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 按建议来 | 确认所有建议 |
| B. 某些条目我想调整 | 逐条讨论 |
| C. 全部让我重新看一遍 | 逐条展示确认 |

---

### CP-⑦：CP3 补充遗漏条目

**时机**：用户确认完锁定项后
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"锁定项已确认！最后确认一下——有没有什么我可能遗漏的、绝对不能改的设定？"

**交互内容**：

```
❓ 还有什么是我可能遗漏的、绝对不能改的设定？
比如：
- 某个角色的隐藏身份
- 某个未揭示的伏笔真相
- 你心中对后续走向的某个执念
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 没有遗漏了 | 进入Phase 7 |
| B. 我要补充 | 用户输入后追加到清单 |
| C. 让我看看原作再确认 | 暂停等待用户 |

---

### CP-⑧：衔接点确认

**时机**：进入 Phase 7 前
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"准备开始规划细纲了！先确认一下续写从哪里接、怎么接。"

**交互内容**：

```
🔗 衔接点确认：

原作最后一章（第{N}章）结尾状态：
- {状态1}
- {状态2}
- {状态3}
- 章末钩子：{钩子}

续写起始方式选择：
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 直接接续最后一章结尾 | 从章末钩子开始 |
| B. 时间跳跃 | 从{N}天后/后开始，先写日常再引入事件 |
| C. 视角切换 | 先写其他角色反应再回到主角 |
| D. 自定义 | 用户指定 |

---

### CP-⑨：首章方向选择

**时机**：Phase 8 生成首章细纲后
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"首章细纲已生成！这一章是续写的开篇，方向很重要。请选择你想要的开局方式。"

**交互内容**：

```
📖 第{N}章方向选择：

A. 紧张开局：{描述}
   适合：{路线类型}，快速进入冲突

B. 悬念开局：{描述}
   适合：{路线类型}，先铺垫再爆发

C. 反差开局：{描述}
   适合：{路线类型}，情绪反差大

❓ 你想用哪种开局？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 紧张开局 | 直接进入冲突 |
| B. 悬念开局 | 先铺垫再爆发 |
| C. 反差开局 | 情绪反差 |
| D. 自定义 | 用户指定 |

---

### CP-⑩：Sample Chapter 讨论

**时机**：Sample Chapter 生成后
**模式**：所有模式必停

**状态摘要**：
"试写片段已完成！请审核风格是否与原作一致。"

**交互内容**：

```
✏️ 试写片段（{字数}字）：

{试写内容}

---

风格对比：
- 句式节奏：{与原作对比} ✅/⚠️/❌
- 对话风格：{与原作对比} ✅/⚠️/❌
- 情绪衔接：{与原作对比} ✅/⚠️/❌
- 用词习惯：{与原作对比} ✅/⚠️/❌

❓ 你觉得这个风格怎么样？哪里需要调整？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 风格可以，继续写完整章节 | 进入正文生成 |
| B. {具体偏差}需要调整 | 收集偏差描述后重新试写 |
| C. 节奏需要调整 | 调整后重新试写 |
| D. 完全不对，重新试写 | 从头重新生成 |

---

### CP-⑪：每章关键情节点确认

**时机**：Phase 10 每章正文生成前
**模式**：所有模式必停（仅首章）/ 讨论模式每章都停 / 快速模式仅首章停

**状态摘要**：
"本章即将生成，请确认关键情节点是否合适。"

**交互内容**：

```
📝 第{N}章即将生成，本章关键点：

1. {情节点1}
2. {情节点2}
3. {情节点3}
4. 章末：{章末钩子}

❓ 这些情节点可以吗？有没有想加/改/删的？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 可以，开始写 | 生成正文 |
| B. 想调整某些点 | 逐点讨论 |
| C. 这章先不写 | 跳过本章 |
| D. 想先讨论某个点 | 深入讨论 |

---

### CP-⑫：审核结果讨论

**时机**：Phase 11 一致性审核和 Phase 12 润色评分后
**模式**：讨论模式必停 / 快速模式仅在评分低于出稿门槛时停

**状态摘要**：
"本章审核完成！让我展示审核结果和具体问题。"

**交互内容**：

```
📊 第{N}章审核结果：

一致性评分：{分数}/10
风格指纹偏离：{超阈值项数}项超阈值（{具体项}）

主要问题：
1. {问题描述}
2. {问题描述}

❓ 你想怎么处理？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 按建议润色 | 执行润色 |
| B. 某些问题我可以接受 | 标注后通过 |
| C. 需要重写某些部分 | 局部重写 |
| D. 重新写这一章 | 回到Phase 10 |

## 衔接话术

决策点之间的自然衔接话术：

| 从 → 到 | 话术 |
|---------|------|
| CP-①→CP-② | "好的，让我逐项确认你的修正意见。" |
| CP-①→CP2 | "提取结果已确认！接下来让我诊断你的卡点。" |
| CP-②→CP2 | "修正完成！接下来进入卡点诊断。" |
| CP-③→CP-④ | "诊断已确认！让我给你准备续写路线。" |
| CP-④→CP-⑤ | "路线已选定！让我展开具体的章节节拍。" |
| CP-⑤→CP3 | "节拍已确认！在开始写之前，我们需要确认不可改动的设定。" |
| CP-⑥→CP-⑦ | "锁定项已确认！最后确认一下有没有遗漏。" |
| CP-⑦→Phase 7 | "所有准备工作完成！让我开始规划续写细纲。" |
| CP-⑧→CP-⑨ | "衔接方式已确认！让我准备首章细纲。" |
| CP-⑨→Phase 10 | "首章方向已确认！让我先试写一段确认风格。" |
| CP-⑩→CP-⑪ | "风格已确认！让我展示首章的关键情节点。" |
| CP-⑪→正文 | "情节点已确认！开始生成正文。" |
| CP-⑫→下一章 | "本章完成！继续下一章。" |

---

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 用户在决策点选择"自定义" | 收集用户描述后调整方案 |
| 用户连续3次选择"不满意" | 建议回退到上一个决策点重新讨论 |
| 用户在快速模式下要求讨论 | 立即切换到讨论模式 |
| 用户跳过决策点 | 记录到 SESSION.md，使用 AI 推荐选项 |
| 用户在决策点提出新需求 | 评估是否需要回退到更早的阶段 |
