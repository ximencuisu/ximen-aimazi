# 半部小说续写与卡文救援

> 何时加载：续写模式

## 1. 概述

续写模式不是绕开主流程，而是在主流程前嵌入 CP1-CP3 准备阶段，复用 Phase 7-12。

核心思路：先从已有正文中反向提取作品 DNA、风格指纹、人物状态、伏笔状态，再诊断卡点、确认不可改动设定，最后无缝衔接主流程的细纲规划和正文生成。

### 续写流程总览

```
用户输入 → 场景识别 → CP1 反向解析 → CP2 卡点诊断与续写方案 → CP3 不可改动设定确认 → Phase 7-12
                         │                    │                        │
                         ▼                    ▼                        ▼
                   作品DNA/风格指纹     三条续写路线+章节拍      Frozen_Canon 门控
                   角色/伏笔/世界观     续写诊断+续写方案        不可改动清单
```

---

## 2. 续写模式入口与场景识别

### 5 种主场景

5 种续写场景（A 卡文诊断 / B 外稿续写 / C 风格漂移修正 / D 旧坑回填 / E 跨作者接力）的编号、触发词与处理流程详见 `continuation-overview.md`"续写场景类型"表。

各场景在 CP1 阶段的特殊侧重（engine 独有补充）：

| 场景 | CP1 侧重 |
|------|---------|
| A 卡文诊断 | 标准 CP1→CP2→CP3→Phase 7-12 |
| B 外稿续写 | 标准 CP1→CP2→CP3→Phase 7-12 |
| C 风格漂移修正 | CP1 侧重风格指纹提取 |
| D 旧坑回填 | CP1 侧重伏笔盘点 |
| E 跨作者接力 | CP1 额外提取标志性句式和叙述视角偏好 |

### 补充子场景

| 子场景 | 父场景 | 条件 | 特殊处理 |
|--------|--------|------|---------|
| 单章续写 | B | 已有完整细纲 | 直接进入 Phase 10 |
| 跨卷续写 | A/B | 当前卷正文已完成 | 走卷级复盘 |
| 断更后续写 | A/B | 最新章节时间戳超过7天 | 走全面上下文重建 |
| 修改后续写 | 任意 | 已有项目中修改前文 | 走影响评估→解冻→Phase 8-12 |

### 触发规则

1. 用户输入包含"卡文""续写""接着写""帮我续""旧坑""填坑""跨作者""接力""风格漂移""味道变了"中任一关键词 → 激活续写模式并询问场景类型
2. 用户提供已有正文文件路径或粘贴正文文本超过2000字 → 激活续写模式并自动进入 CP1
3. 用户未提供任何已有正文且未明确说明场景 → 提示用户至少提供原作正文摘要或正文片段
4. 场景类型选择后记录到 `SESSION.md`

### 交互模式

续写流程复用全流程交互模式，用户可随时切换：

| 模式 | 特点 | 适合用户 |
|------|------|---------|
| 🗨️ 讨论模式（默认） | 每个决策点都展示选项+讨论 | 想深度参与方向的作者 |
| ⚡ 快速模式 | 只在关键决策点暂停，其余自动推进 | 想快速出稿的作者 |

**切换方式**：用户说"切换到快速模式"/"切换到讨论模式"

**快速模式必停点**：详见 `continuation-overview.md`"快速模式 4 个必停点"表（CP-④路线选择 / CP-⑥不可改动清单确认 / CP-⑩Sample Chapter确认 / CP-⑪首章关键情节点确认）

> 完整决策点交互模板见 `references/continuation-interaction.md`，主流程决策点见 `references/workflow-interaction.md`。

## 3. CP1 反向解析阶段

**目标**：从已有正文中系统化提取作品 DNA、风格指纹、人物状态、伏笔状态、世界观规则

**输入**：用户提供的全部原作素材（Continuation_Source_Input），可能是正文章节文件、纯文本粘贴、或带原作+自己续写半截的混合素材

### 产出物

| 文件 | 内容 |
|------|------|
| `设定/作品DNA.md` | 类型承诺、主角驱动力、主线问题、副线结构、读者爽点契约 |
| `设定/风格指纹.md` | 9项可量化指标（见下） |
| `设定/角色/{角色名}.md` | 外貌、性格、关系、当前状态、所在地、持有资源 |
| `追踪/伏笔.md` | 埋设章节、状态（已埋未收/部分回收/已回收）、相关角色 |
| `设定/世界观/背景设定.md` | 从原作归纳 |
| `设定/世界观/力量体系.md` | 等级、边界、代价 |
| `.learnings/` 全部档案 | 从原作回填 |

### 风格指纹 9 项指标

| 指标 | 说明 | 漂移阈值 |
|------|------|---------|
| 平均句长 | 字符数均值 | ±20% |
| 句长分布区间 | 短/中/长句比例 | ±20% |
| 平均段长 | 行数均值 | ±25% |
| 对话占比 | 对话行数/总行数 | ±15% |
| 高频意象前20 | 出现频率最高的意象词 | 重合度≥60% |
| 高频动词前20 | 出现频率最高的动作词 | 重合度≥80% |
| 句式偏好 | 陈述/疑问/感叹/倒装比例 | ±20% |
| 动作-对话-情绪比例 | 三类描写占比 | ±20% |
| 原作回避表达倾向 | 原作不用的表达方式 | 0（不得出现） |

### 特殊处理

- **原文<5000字** → 提示置信度不足，询问是否继续
- **原作存在前后矛盾** → 列出矛盾清单，用户裁决
- **跨作者接力（E）** → 额外提取标志性句式和叙述视角偏好
- **`.learnings/` 已有内容** → 逐条比对，用户裁决"保留/覆盖/合并"

### Metadata 规则

CP1 产出的每个文件在开头附加 metadata 块：

```yaml
---
来源: 原作提取
生成时间: YYYY-MM-DD HH:mm
所属 Scene_Type: A/B/C/D/F
所属阶段: CP1
源文件清单: [文件列表]
---
```

### 完成条件

全部产出文件生成完毕 → 进入CP-①提取结果审核：

1. 向用户展示反向解析摘要（作品DNA/风格指纹/角色状态/伏笔盘点）
2. 询问用户"这些理解对吗？有没有理解错或遗漏的地方？"
3. 用户确认/修正后 → 进入 CP2

如用户选择修正 → 进入CP-②关键设定修正，逐项确认修正内容。

> 决策点交互模板见 `references/continuation-interaction.md`。

### 输入素材处理

#### 素材类型

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| 正文章节文件 | 用户提供的 .md 或 .txt 文件 | 逐文件读取，按文件名排序确定章节顺序 |
| 纯文本粘贴 | 用户直接粘贴的正文内容 | 合并为连续文本，按自然分段识别章节 |
| 混合素材 | 原作 + 用户自己续写半截的内容 | 识别分界点，分别标记来源 |

#### 样本量检查

| 字数 | 处理方式 |
|------|---------|
| < 5000 字 | 提示"原文样本量偏低，风格指纹置信度可能不足"，询问是否继续 |
| 5000-30000 字 | 正常处理，风格指纹置信度中等 |
| > 30000 字 | 正常处理，风格指纹置信度较高 |

### 产出物提取方法

#### 1. 作品DNA提取

从正文中提取以下五项：

| 项目 | 提取方法 | 示例 |
|------|---------|------|
| 类型承诺 | 识别题材类型和读者期待 | "玄幻升级流，承诺读者看主角从废材到巅峰" |
| 主角驱动力 | 提取主角的核心目标和情感动机 | "为父复仇→证明自己→守护身边人" |
| 主线问题 | 识别全书要解决的核心矛盾 | "主角能否在三年之约中战胜对手" |
| 副线结构 | 梳理情感线、势力线、悬念线等 | "感情线：与女主从敌对到并肩；势力线：从散修到宗门核心" |
| 读者爽点契约 | 归纳读者追这本书期待什么爽感 | "装逼打脸+升级碾压+身份揭秘" |

**矛盾检测**：如果正文中存在前后矛盾的设定（如角色能力前后不一致、时间线冲突），在 `设定/作品DNA.md` 中列出矛盾清单并提示用户人工裁决。

#### 2. 风格指纹提取

##### 九项指标计算方法

| 指标 | 计算方法 | 单位 |
|------|---------|------|
| 平均句长 | 全文按句号/问号/叹号分句，计算字符数均值 | 字符/句 |
| 句长分布区间 | 短句(≤15字)/中句(16-35字)/长句(≥36字) 的百分比 | 三段比例 |
| 平均段长 | 按空行分段，计算每段行数均值 | 行/段 |
| 对话占比 | 引号内文本行数 / 全文总行数 | 百分比 |
| 高频意象前20 | 对全文做名词词频统计，取出现频率最高的20个具象名词 | 词表+频率 |
| 高频动词前20 | 对全文做动词词频统计，取出现频率最高的20个动作词 | 词表+频率 |
| 句式偏好 | 陈述/疑问/感叹/倒装各自的占比 | 四类比例 |
| 动作-对话-情绪比例 | 动作描写行/对话行/情绪描写行各自的占比 | 三类比例 |
| 原作回避表达倾向 | 全文搜索常见AI味词汇，标记原作完全不使用的表达 | 词表 |

##### 提取流程

1. 逐文件读取素材，合并为连续文本
2. 按上述9项指标分别计算
3. 将结果写入 `设定/风格指纹.md`，格式见 `references/style-fingerprint-guide.md`

#### 3. 角色状态提取

从最新章节向前扫描，提取每个出场角色的：

| 维度 | 提取方法 |
|------|---------|
| 当前等级/实力 | 从最近战斗/突破描写中提取 |
| 所在地 | 从最近场景描写中提取 |
| 持有资源 | 从最近获得/使用物品描写中提取 |
| 人际关系 | 从最近互动描写中提取 |
| 情感状态 | 从最近心理/行为描写中提取 |

#### 4. 伏笔盘点

扫描全文，识别以下类型的伏笔：

| 伏笔类型 | 识别标志 |
|---------|---------|
| 悬念类 | 角色提出疑问但未解答、出现神秘人物/物品 |
| 承诺类 | 角色做出承诺/计划但未执行 |
| 预兆类 | 环境/对话暗示未来事件 |
| 冲突类 | 埋下矛盾种子但未爆发 |

每条伏笔记录：埋设章节、内容摘要、当前状态（已埋未收/部分回收/已回收）、相关角色。

#### 5. 世界观规则提取

从正文中归纳：

| 维度 | 提取方法 |
|------|---------|
| 力量体系边界 | 从战斗描写中提取各等级的能力上限和代价 |
| 社会规则 | 从势力互动中提取权力结构和行为准则 |
| 经济系统 | 从交易/资源描写中提取货币体系和物价水平 |
| 地理格局 | 从移动/场景描写中提取空间关系 |

## 4. CP2 卡点诊断与续写方案阶段

**目标**：基于反向解析结果诊断卡点，给出三条续写路线

**输入**：CP1 产出的全部档案 + 用户提供的卡点描述

### 卡点归类（5类）

| 卡点类型 | 表现 | 典型解法 |
|------|------|------|
| 结构卡 | 不知道下一卷写什么 | 重立阶段目标和反派压力 |
| 人物卡 | 主角没欲望、配角没作用 | 重写角色短期目标和关系冲突 |
| 资源卡 | 主角太强或太穷都写不动 | 调整资源账本与代价机制 |
| 信息卡 | 真相揭太多或什么都没揭 | 重新安排信息释放节奏 |
| 情绪卡 | 剧情在走，但读者没感觉 | 增加情感债、羞辱债、承诺债 |

### 产出物

| 文件 | 内容 |
|------|------|
| `设定/续写诊断.md` | Scene_Type、卡点归类、根因分析、与作品DNA偏离评估 |
| `设定/续写方案.md` | 三条路线（稳妥/提速/爆点）+ 每条路线3-10章节拍 |

### 三条路线要求

- **稳妥线**：最贴近前文，风险低
- **提速线**：快速给爽点，适合救节奏
- **爆点线**：引入更大转折，适合中后期抬盘

每条路线必须说明：核心事件、为什么现在发生、对主线推进、引出的新冲突

每条路线的3-10章节拍包含：章节定位、核心事件、出场角色、章末钩子

### 场景特殊处理

- **旧坑回填（D）** → 每条路线明确指明回收哪些未回收伏笔、回收章节位置
- **风格漂移修正（C）** → 识别偏离风格指纹的段落范围，标注需要回拉的段落
- **路线不满意** → 最多迭代3轮，第3轮后建议回退CP1补充原作信息

### 完成条件

1. 卡点归类完成 → 进入CP-③卡点诊断讨论，让用户确认诊断是否准确
2. 诊断确认后展示三条路线 → 进入CP-④路线选择与调整（必停点）
3. 路线选定后展示3-10章节拍 → 进入CP-⑤节拍审核
4. 节拍确认后 → 进入 CP3

> 决策点交互模板见 `references/continuation-interaction.md`。

## 5. CP3 不可改动设定清单确认（强门控）

**目标**：在AI开始写续写正文之前，与用户共同确认"不可改动设定清单"

**产出物**：`设定/不可改动清单.md`

### 五类条目

1. 世界观规则
2. 力量体系边界
3. 主要角色当前状态
4. 已发生不可逆事件
5. 已埋未收伏笔

### 门控规则

- 基于 CP1 产出物自动生成草稿，每条附带锁定理由
- 进入CP-⑥锁定理由展示，让用户确认锁定建议是否合理（必停点）
- 用户逐项确认"锁定/可调整/删除"
- 确认完毕后进入CP-⑦补充遗漏条目，询问用户是否有遗漏
- 所有"锁定"条目标记为 Frozen_Canon 写入 Frozen 区
- **Frozen_Canon 中任何条目尚未被用户确认 → 拒绝进入 Phase 7**
- 续写过程中（Phase 8/10/11）检测到与 Frozen_Canon 冲突 → 中断当前生成，列出冲突条目，用户选择"修改 Frozen_Canon"或"修改细纲/正文"
- 修改 Frozen_Canon → 追加变更记录（时间、原条目、新条目、变更原因）+ 二次确认
- **跨作者接力（E）** → 额外锁定叙述视角和标志性句式偏好

### 完成条件

Frozen_Canon 区非空且所有条目已确认 → 可进入 Phase 7

## 6. 续写专用 Phase 7-10 衔接

- CP3 完成后进入CP-⑧衔接点确认，让用户选择续写起始方式（直接接续/时间跳跃/视角切换/自定义）
- 衔接方式确认后跳过 Phase 1-6，直接进入 Phase 7
- Phase 8 生成首章细纲后进入CP-⑨首章方向选择，展示3种开局方式供用户选择
- Phase 7 自动加载 `设定/作品DNA.md`、`设定/风格指纹.md`、`设定/不可改动清单.md` 作为细纲规划上下文
- CP2 产出的"未来3-10章节拍"作为细纲基线
- Phase 8 每章细纲标注"本章涉及的 Frozen_Canon 条目"清单
- Phase 10 首次生成正文前先写 Sample_Chapter（500-1000字试写），产出 `正文/续写试写_{时间戳}.md`
- 进入CP-⑩Sample Chapter讨论（必停点），展示试写内容+风格对比，让用户具体讨论哪里需要调整
- Phase 10 每章正文生成前进入CP-⑪每章关键情节点确认，展示本章关键点供用户审核（首章必停，后续章节讨论模式停/快速模式跳过）
- 续写章节编号与原作保持同序连号
- 续写章节 metadata 标注来源为 `AI续写` + 引用的 Frozen_Canon 条目

## 7. 风格指纹比对与漂移控制

### Phase 11 增强

- 计算当前续写章节相对于 `设定/风格指纹.md` 的9项指标偏离度
- 产出 `追踪/续写风格对比_第{N}章.md`，包含每项指标的原作基线、当前章节实测值、偏离百分比
- 任意指标超过漂移阈值 → 标记"超阈值"
- **3项以上指标超阈值 → 阻断 Phase 12**，用户决定"重写/调整阈值/接受偏离"
- 审核完成后进入CP-⑫审核结果讨论，展示具体问题并让用户决定处理方式

### Phase 12 增强

- 润色阶段额外执行风格指纹回拉规则，把超阈值的句长、段长、对话占比、高频意象重新拉向原作基线
- 润色后重新计算9项指标，追加"润色后"列

## 8. 记忆系统双向同步

- CP1 完成后从原作回填 `.learnings/` 全部档案（STORY_BIBLE/CHARACTERS/LOCATIONS/PLOT_POINTS/SUSPENSE/RESOURCES/SUBPLOTS/EMOTIONS）
- 已有内容逐条裁决"保留/覆盖/合并"
- 每条写入项追加 metadata 标注来源为 `原作提取`
- Phase 10 后按现有规则更新 `.learnings/`，标注来源为 `AI续写`
- 用户手动修改 → 标注来源为 `用户修改`
- `追踪/CHAPTERS.md` 区分原作/续写章节，追加来源标记列
- 续写章节生成完毕 → 更新 SESSION.md 中 Continuation_Mode 状态、Scene_Type、最近风格对比偏离值

## 9. 产出物 Metadata 与来源追溯

每个续写流程文件开头附加 metadata 块：

```yaml
---
来源: 原作提取 / AI生成 / AI续写 / 用户修改
生成时间: YYYY-MM-DD HH:mm
所属 Scene_Type: A/B/C/D/F
所属阶段: CP1/CP2/CP3/Phase7/Phase8/Phase10/Phase11/Phase12
变更历史:
  - 时间 | 来源 | 摘要
---
```

## 10. 续写技法

### 文风一致性技法

| 技法 | 操作要点 |
|------|---------|
| 文风指纹对照法 | 续写前先从最近3章提取风格特征，续写后逐项比对9项指标 |
| 句式节奏复现 | 分析前文的短句/长句交替模式，续写时保持相同节奏 |
| 对话风格延续 | 每个角色的口头禅、说话习惯、语气特征必须与前文一致 |

### 情绪衔接技法

| 技法 | 操作要点 |
|------|---------|
| 情绪承接法 | 续写开头必须承接上一章结尾的情绪状态，不能突然跳转 |
| 钩子兑现法 | 上一章章末钩子必须在续写开头3段内回应 |
| 节奏延续法 | 分析前文是快节奏还是慢节奏，续写前500字保持相同节奏再逐步调整 |

### 前文坑处理技法

| 技法 | 操作要点 |
|------|---------|
| 伏笔优先级排序 | 按"读者记忆度×剧情紧迫度"排序，优先处理高优先级伏笔 |
| 自然回收法 | 不刻意"填坑"，而是在新剧情推进中自然带出旧伏笔的回收 |
| 延迟回收法 | 如果当前不适合回收，给一个"reminder"维持热度 |

### 创新与延续平衡技法

| 技法 | 操作要点 |
|------|---------|
| "70%延续+30%创新"原则 | 续写内容70%延续前文风格和节奏，30%引入新元素 |
| 新元素植入法 | 新角色/新设定/新冲突必须通过已有角色的视角引入 |
| 避免推翻法 | 续写中绝不否定前文已确立的事实，只能补充和扩展 |

## 11. 续写一致性检查清单

| 维度 | 检查项 | 权重 |
|------|--------|------|
| 文风一致性 | 句式节奏、用词习惯、描写风格是否与前文一致 | 25% |
| 角色一致性 | 角色行为、对话口吻、决策逻辑是否符合人设 | 25% |
| 情绪衔接性 | 续写开头是否自然承接前文结尾情绪 | 15% |
| 伏笔管理 | 是否遗漏了该回收的伏笔、是否埋设了新伏笔 | 15% |
| 节奏延续性 | 爽点间隔、高潮周期是否与前文节奏匹配 | 10% |
| 创新适度性 | 新元素是否自然融入、是否推翻前文设定 | 10% |

**续写出稿门槛**：总分 ≥ 7.5

## 12. 续写中断、回退与异常处理

- 任意阶段输入"中断续写""退出续写""暂停" → 保存 Continuation_Mode 状态到 SESSION.md 并退出
- 再次触发续写且 SESSION.md 有未完成状态 → 提示"检测到上次续写流程在 {阶段}，是否恢复"
- CP3后回退CP1 → 列出会被覆盖的产出物清单 + 二次确认
- 输入文件不存在或无法读取 → 终止CP1并提示
- 原作乱码/非小说内容/与描述题材严重不符 → 终止解析+报告问题样本
- 产出文件写入失败 → 记录 `.learnings/ERRORS.md` + 向用户报告
- 重大产出物写入前备份到 `设定/.backup/{阶段}_{时间戳}/`

## 13. 常见卡文类型与补救

### 1. 主角刚突破，突然没事干

- 给更高层级的资源需求或身份压力
- 让突破立即带来新的敌意、机会或曝光
- 不要让升级停留在庆祝层面

### 2. 反派压不住了

- 给反派明确筹码，而不是只给嘴炮
- 让反派掌控规则、资源、情报其中至少一种
- 反派最好能伤到主角在乎的东西

### 3. 支线太多，主线散了

- 明确"当前卷唯一主问题"
- 支线必须服务主问题，否则休眠
- 把休眠支线写入 `SUBPLOTS.md`

### 4. 女角色很多，但都像挂件

- 每个重要角色必须有独立目标
- 给她们自己的资源、风险和决策权
- 关系推进要改变剧情，不只是调情

### 5. 爽点越来越麻

- 不只升级数值，要升级代价、身份、局势
- 从"打小怪"转向"改变规则"
- 用羞辱回收、旧敌复盘、身份翻转做新刺激

### 6. 换地图后读者流失

- 旧角色一刀切/新设定一次倒出/新旧地图无关联 → 三种致命操作
- 解法：旧角色至少保留1-2人过渡、新设定分3章以上逐步展开、新旧地图通过资源/人脉/威胁建立关联
- 换地图不是重启，是升级——旧积累必须在新地图有投射

### 7. 金手指僵化

- 后期只剩品质提升一个维度，提升感消失
- 解法：给金手指增加新维度（功能拓展/使用代价/限制条件/进化分支）
- 让金手指从"工具"变成"需要管理的资源"，引入使用成本和副作用
- 金手指升级应该改变玩法，不是只改数值

### 8. 感情线写死了

- 确认关系后无聊，需要持续制造新的感情张力
- 解法：外部压力（家族/势力/身份对立）、内部矛盾（价值观/目标冲突）、第三方介入（不是三角恋，而是利益绑定）
- 感情线不是终点，是新的冲突起点——确认关系后应该解锁新的矛盾维度
- 每次关系推进必须带来新的剧情后果

## 14. 与现有系统的协作关系

### 基础协作

- 续写前先读 `SESSION.md`
- 如果续写涉及新设定，同步写入 `STORY_BIBLE.md`
- 如果续写改变子剧情状态，同步更新 `SUBPLOTS.md`
- 如果回收或新增悬念，同步更新 `PLOT_SUSPENSE.md`
- 如果调整资源与战力，更新 `RESOURCES.md`

续写模式不是绕开 `ximen-aimazi` 的流程，而是在已有长篇系统上做"卡文救援"。

### CP1 产出物与 `.learnings/` 的映射关系

| CP1 产出物 | 对应 `.learnings/` 档案 |
|------------|------------------------|
| `设定/作品DNA.md` | `STORY_BIBLE.md` |
| `设定/角色/{角色名}.md` | `CHARACTERS.md` |
| `设定/世界观/背景设定.md` | `LOCATIONS.md` |
| `设定/世界观/力量体系.md` | `STORY_BIBLE.md` + `RESOURCES.md` |
| `追踪/伏笔.md` | `PLOT_SUSPENSE.md` |

### Frozen_Canon 与 Phase 8/10/11 的冲突检测机制

- Phase 8 生成细纲时：每章标注涉及的 Frozen_Canon 条目，自动比对是否冲突
- Phase 10 生成正文前：检查本章细纲涉及的 Frozen_Canon 条目，确认无冲突后生成
- Phase 11 一致性审核时：额外检查正文内容是否违反 Frozen_Canon 条目
- 冲突发现 → 中断 + 列出冲突条目 + 用户裁决

### 风格指纹比对与 Phase 11/12 的集成方式

- Phase 11：在原有7维一致性审核基础上，追加9项风格指纹偏离度计算，产出风格对比报告
- Phase 12：润色阶段将风格回拉作为额外润色维度，润色后重新计算9项指标验证
