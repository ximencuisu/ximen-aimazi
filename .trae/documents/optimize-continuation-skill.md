# 优化网文写作 Skill 续写功能 — 实施计划（v2 整合版）

## 一、现有续写功能分析

### 当前已有内容

1. **`continuation-engine.md`**：六步续写引擎（DNA拆解→事实校准→卡点诊断→三条路线→章节拍→试写开口）+ 5种常见卡文类型 + 输出文件 + 系统协作关系
2. **SKILL.md 增强模式**：仅一行触发条件 `"卡文了""后面不会写了"` → 查阅 continuation-engine.md
3. **项目进度检测表**：5种检测标志判断恢复位置
4. **半部续写说明**：先走 Phase 1-4 快速补齐设定，再根据进度检测进入对应阶段

### 核心缺陷

| 缺陷 | 说明 |
|------|------|
| 续写场景分类太粗 | 只有"卡文救援"和"半部续写"，缺少外稿续写、风格漂移修正、旧坑回填、跨作者接力等场景 |
| 缺乏系统化反向解析 | 事实校准只列了检查项，没有从已有正文反向提取作品DNA、风格指纹、人物状态、伏笔状态的产出与产物 |
| 缺乏不可改动设定清单 | 没有"强门控"确认机制，续写过程中可能无意推翻原作设定 |
| 缺乏风格一致性可验证比对 | 没有量化风格指纹和漂移阈值，无法客观判断续写是否偏离原作风格 |
| 续写与主流程衔接不清 | 续写模式如何回到正常流程、续写中的批次细纲如何处理、续写后的记忆同步规则缺失 |
| 缺乏具体续写技法 | 只有诊断和路线，没有"如何保持前后文风一致""如何自然衔接情绪""如何处理前文坑"等实操技法 |
| 缺乏产出物来源追溯 | 续写产出物融入主目录后无法区分原作提取/AI生成/用户修改/AI续写 |
| 缺乏中断回退机制 | 续写流程无法安全中断、回退或恢复 |
| SKILL.md 中续写描述太弱 | 增强模式只有一行，项目进度检测不够细致，缺少续写专用子流程 |
| 触发条件太少 | 只识别"卡文了""后面不会写了"，不识别"续写""接着写""旧坑""填坑""跨作者"等 |

## 二、整合方案（合并 kiro specs 需求 + 原计划）

### 2.1 续写场景分类（整合版）

采用 kiro specs 的 5 种主场景 + 原计划的补充场景：

| 编号 | 场景类型 | 触发词/条件 | 处理方式 |
|------|---------|------------|---------|
| A | 卡文诊断 | "卡文""写不下去了""不知道怎么写" | CP1→CP2→CP3→Phase 7-12 |
| B | 外稿续写 | 提供已有正文>2000字、"续写""接着写""帮我续" | CP1→CP2→CP3→Phase 7-12 |
| C | 风格漂移修正 | "风格漂移""味道变了""不像我写的" | CP1(侧重风格指纹)→CP2→CP3→Phase 7-12 |
| D | 旧坑回填 | "旧坑""填坑""伏笔没回收" | CP1(侧重伏笔盘点)→CP2→CP3→Phase 7-12 |
| F | 跨作者接力 | "跨作者""接力""接手别人作品" | CP1(额外提取标志性句式)→CP2→CP3→Phase 7-12 |

补充子场景（在主场景内按条件触发）：
- 单章续写：B场景下，已有完整细纲时直接进入 Phase 10
- 跨卷续写：A/B场景下，当前卷正文已完成时走卷级复盘
- 断更后续写：A/B场景下，最新章节时间戳超过7天时走全面上下文重建
- 修改后续写：已有项目中修改前文后，走影响评估→解冻→Phase 8-12

### 2.2 CP1 反向解析阶段（新增，替代原"事实校准"）

**产出物**：

| 文件 | 内容 | 来源标记 |
|------|------|---------|
| `设定/作品DNA.md` | 类型承诺、主角驱动力、主线问题、副线结构、读者爽点契约 | `原作提取` |
| `设定/风格指纹.md` | 9项可量化指标（见下） | `原作提取` |
| `设定/角色/{角色名}.md` | 外貌、性格、关系、当前状态、所在地、持有资源 | `原作提取` |
| `追踪/伏笔.md` | 埋设章节、状态（已埋未收/部分回收/已回收）、相关角色 | `原作提取` |
| `设定/世界观/背景设定.md` | 从原作归纳 | `原作提取` |
| `设定/世界观/力量体系.md` | 等级、边界、代价 | `原作提取` |
| `.learnings/` 全部档案 | 从原作回填，已有内容时逐条裁决 | `原作提取` |

**风格指纹 9 项指标**：

| 指标 | 说明 | 漂移阈值 |
|------|------|---------|
| 平均句长 | 字符数均值 | ±20% |
| 句长分布区间 | 短/中/长句比例 | ±20% |
| 平均段长 | 行数均值 | ±25% |
| 对话占比 | 对话行数/总行数 | ±15% |
| 高频意象前20 | 出现频率最高的意象词 | 重合度≥60% |
| 高频动词前20 | 出现频率最高的动作词 | ±20% |
| 句式偏好 | 陈述/疑问/感叹/倒装比例 | ±20% |
| 动作-对话-情绪比例 | 三类描写占比 | ±20% |
| 原作回避表达倾向 | 原作不用的表达方式 | 0（不得出现） |

**特殊处理**：
- 原文<5000字 → 提示置信度不足
- 原作存在前后矛盾 → 列出矛盾清单，用户裁决
- 跨作者接力（F）→ 额外提取标志性句式和叙述视角偏好
- `.learnings/` 已有内容 → 逐条比对，用户裁决"保留/覆盖/合并"

### 2.3 CP2 卡点诊断与续写方案阶段（增强原六步引擎）

**产出物**：

| 文件 | 内容 |
|------|------|
| `设定/续写诊断.md` | Scene_Type、卡点归类（结构/人物/资源/信息/情绪）、根因分析、与作品DNA偏离评估 |
| `设定/续写方案.md` | 三条路线（稳妥/提速/爆点）+ 每条路线3-10章节拍 |

**场景特殊处理**：
- 旧坑回填（D）→ 每条路线明确指明回收哪些未回收伏笔、回收章节位置
- 风格漂移修正（C）→ 识别偏离风格指纹的段落范围
- 路线不满意 → 最多迭代3轮，第3轮后建议回退CP1

### 2.4 CP3 不可改动设定清单确认（新增，强门控）

**产出物**：`设定/不可改动清单.md`

**五类条目**：世界观规则、力量体系边界、主要角色当前状态、已发生不可逆事件、已埋未收伏笔

**门控规则**：
- 用户逐项确认"锁定/可调整/删除"
- 未确认完毕 → 拒绝进入 Phase 7
- 续写过程中与 Frozen_Canon 冲突 → 中断生成，用户选择"修改清单/修改内容"
- 修改 Frozen_Canon → 追加变更记录 + 二次确认
- 跨作者接力（F）→ 额外锁定叙述视角和标志性句式

### 2.5 续写专用 Phase 7-10 衔接（新增）

- CP3 完成后跳过 Phase 1-6，直接进入 Phase 7
- Phase 7 自动加载作品DNA+风格指纹+不可改动清单
- Phase 8 每章细纲标注"本章涉及的 Frozen_Canon 条目"
- Phase 10 首次生成正文前先写 Sample_Chapter（500-1000字试写）
- 续写章节编号与原作保持同序连号

### 2.6 风格指纹比对与漂移控制（新增，改造 Phase 11/12）

- Phase 11 增加风格指纹比对维度
- 产出 `追踪/续写风格对比_第{N}章.md`
- 3项以上指标超阈值 → 阻断 Phase 12，用户决定
- Phase 12 额外执行风格指纹回拉规则
- 润色后重新计算指标，追加"润色后"列

### 2.7 记忆系统双向同步（新增）

- CP1 完成后从原作回填 `.learnings/` 全部档案
- 已有内容逐条裁决
- 每条写入项追加 metadata 标注来源
- Phase 10 后按现有规则更新，标注 `AI续写`
- `追踪/CHAPTERS.md` 区分原作/续写章节

### 2.8 产出物 Metadata 与来源追溯（新增）

每个续写流程文件开头附加 metadata 块：
```
---
来源: 原作提取 / AI生成 / AI续写 / 用户修改
生成时间: YYYY-MM-DD HH:mm
所属 Scene_Type: A/B/C/D/F
所属阶段: CP1/CP2/CP3/Phase7/Phase8/Phase10/Phase11/Phase12
变更历史:
  - 时间 | 来源 | 摘要
---
```

额外产出 `设定/续写产物索引.md` 列出所有 AI 续写/AI 生成来源的文件。

### 2.9 续写技法（保留原计划，作为参考指南）

4大类12项技法保留在 continuation-engine.md 中：
1. 文风一致性技法（指纹对照/句式复现/对话延续）
2. 情绪衔接技法（情绪承接/钩子兑现/节奏延续）
3. 前文坑处理技法（优先级排序/自然回收/延迟回收）
4. 创新与延续平衡技法（70/30原则/新元素植入/避免推翻）

### 2.10 续写中断、回退与异常处理（新增）

- 任意阶段可输入"中断续写""退出续写""暂停" → 保存状态到 SESSION.md
- 再次触发续写 → 检测未完成状态，提示恢复
- CP3后回退CP1 → 列出覆盖清单 + 二次确认
- 输入文件不存在 → 终止CP1
- 原作乱码/非小说内容 → 终止解析 + 报告问题样本
- 写入失败 → 记录 ERRORS.md + 报告
- 重大产出物写入前备份到 `设定/.backup/{阶段}_{时间戳}/`
