# 续写交互增强计划 — 让用户掌握续写方向

## 一、问题分析

### 当前交互点（7个，偏少且粒度粗）

| 阶段 | 当前交互 | 问题 |
|------|---------|------|
| 入口 | 选择场景类型 A/B/C/D/F | ✅ 够用 |
| CP1 | 仅"是否继续"和"是否进入CP2" | ❌ 用户无法审核/修正提取结果 |
| CP2 | 选一条路线或自定义 | ❌ 缺少诊断讨论、节拍审核 |
| CP3 | 逐项确认"锁定/可调整/删除" | ❌ 缺少为什么锁定的解释、缺少补充入口 |
| Phase 10 | Sample Chapter 确认 | ❌ 缺少章节开头方式选择、关键情节点确认 |
| Phase 11 | 风格漂移三选一 | ⚠️ 交互太少，只有超阈值才触发 |
| Frozen_Canon 冲突 | 二选一 | ✅ 够用 |

### 核心问题

1. **CP1 是"黑箱"**：AI 提取完就问"进不进下一步"，用户没有机会审核和修正提取结果
2. **CP2 是"菜单"**：只给三条路线让用户选，没有讨论过程，用户无法理解为什么是这三条
3. **CP3 是"清单"**：只逐项确认，没有解释为什么建议锁定，用户无法补充遗漏
4. **Phase 7-10 是"自动流水线"**：续写特有的决策点（从哪接、怎么接、接什么）没有交互
5. **整体缺少"讨论感"**：流程是 AI 推进→用户确认，不是 AI 和用户一起讨论

---

## 二、优化方案：在续写流程中嵌入 12 个交互决策点

### 总览

```
入口 → 场景确认
  ↓
CP1 → ①提取结果审核 → ②关键设定修正
  ↓
CP2 → ③卡点诊断讨论 → ④路线选择与调整 → ⑤节拍审核
  ↓
CP3 → ⑥锁定理由展示 → ⑦补充遗漏条目
  ↓
Phase 7-8 → ⑧衔接点确认 → ⑨首章方向选择
  ↓
Phase 10 → ⑩Sample Chapter 讨论 → ⑪每章关键情节点确认
  ↓
Phase 11-12 → ⑫审核结果讨论
```

### 决策点 ①：CP1 提取结果审核

**时机**：CP1 全部产出文件生成完毕后

**交互方式**：展示反向解析摘要，逐模块询问用户

```
📋 反向解析完成！以下是我从原作中提取的核心信息：

【作品DNA】
- 类型承诺：玄幻升级流，承诺读者看主角从废材到巅峰
- 主角驱动力：为父复仇→证明自己→守护身边人
- 主线问题：主角能否在三年之约中战胜对手
- 读者爽点契约：装逼打脸+升级碾压+身份揭秘

❓ 这些理解对吗？有没有我理解错的地方？
```

**选项**：
- A. 基本正确，继续
- B. 有几处需要修正（用户指出）
- C. 遗漏了重要信息（用户补充）
- D. 重新解析，我提供更多原文

### 决策点 ②：CP1 关键设定修正

**时机**：用户在 ① 中选择了 B 或 C

**交互方式**：针对用户指出的修正项，逐项确认

```
你提到主角驱动力不只是"为父复仇"，还有"对母亲的愧疚"。
让我确认修正后的版本：

- 主角驱动力：为父复仇+对母亲的愧疚→证明自己→守护身边人

❓ 这样修正对吗？
```

**选项**：
- A. 对，就这样
- B. 还需要调整
- C. 跳过修正，用原版

### 决策点 ③：CP2 卡点诊断讨论

**时机**：卡点归类完成后，给路线之前

**交互方式**：先展示诊断结果，让用户确认卡点归类

```
🔍 卡点诊断结果：

你的卡点属于【结构卡+情绪卡】复合型：
- 结构卡：主角刚突破金丹期，但下一阶段目标不明确
- 情绪卡：连续3章没有让读者"爽"的情绪节点

根因分析：突破后的"真空期"——旧目标已达成，新压力尚未建立

❓ 这个诊断符合你的感受吗？还是你觉得卡在其他地方？
```

**选项**：
- A. 诊断准确，给路线吧
- B. 主要是结构卡，情绪还好
- C. 我觉得更像是人物卡
- D. 诊断不对，让我描述一下我的感受

### 决策点 ④：CP2 路线选择与调整

**时机**：三条路线展示后

**交互方式**：不只选路线，还讨论每条路线的利弊

```
📌 三条续写路线：

【稳妥线】新地图探索+资源需求
  优点：贴近前文风格，风险低
  风险：可能节奏偏慢，读者觉得"换皮"
  
【提速线】立即引入新反派+紧迫倒计时
  优点：节奏快，爽点密集
  风险：反派铺垫不足可能显得突兀
  
【爆点线】金手指异变+身份暴露危机
  优点：大转折，读者惊喜
  风险：可能推翻已建立的设定平衡

❓ 你倾向哪条？也可以混合，比如"稳妥线节奏+爆点线事件"
```

**选项**：
- A. 稳妥线
- B. 提速线
- C. 爆点线
- D. 混合（用户说明怎么混）
- E. 都不满意，重新给路线

### 决策点 ⑤：CP2 节拍审核

**时机**：用户选定路线后，展示3-10章节拍

**交互方式**：逐章展示节拍，用户可以调整

```
📝 未来5章节拍（提速线）：

第31章「新敌来袭」
  - 核心事件：金丹庆功宴被神秘来客打断
  - 出场角色：主角、新反派、师父
  - 章末钩子：来客揭示主角身世的一角
  ❓ 这一章的方向可以吗？

第32章「身世之谜」
  ...
```

**选项**：
- A. 整体可以，继续
- B. 某几章需要调整（用户指出）
- C. 节奏太快/太慢，整体调
- D. 想加/删某些章节

### 决策点 ⑥：CP3 锁定理由展示

**时机**：展示不可改动清单时

**交互方式**：每条建议锁定的条目都附带理由

```
🔒 不可改动设定清单（建议锁定项）：

1. [建议锁定] 主角已突破金丹期
   理由：已在前文多次确认，推翻会导致读者信任崩塌
   
2. [建议锁定] 师父已战死
   理由：已发生不可逆事件，复活会削弱死亡的意义
   
3. [建议可调整] 主角与女主的关系状态
   理由：目前关系暧昧，续写中可以推进或制造波折

❓ 你觉得这些锁定建议合理吗？
```

**选项**：
- A. 按建议来
- B. 某些条目我想调整（用户指出）
- C. 全部让我重新看一遍

### 决策点 ⑦：CP3 补充遗漏条目

**时机**：用户确认完锁定项后

**交互方式**：询问用户是否有遗漏

```
❓ 还有什么是我可能遗漏的、绝对不能改的设定？
比如：
- 某个角色的隐藏身份
- 某个未揭示的伏笔真相
- 你心中对后续走向的某个执念
```

**选项**：
- A. 没有遗漏了
- B. 我要补充（用户输入）
- C. 让我看看原作再确认

### 决策点 ⑧：衔接点确认

**时机**：进入 Phase 7 前

**交互方式**：确认续写从哪里接、怎么接

```
🔗 衔接点确认：

原作最后一章（第30章）结尾状态：
- 主角刚突破金丹期，正在庆功
- 师父战死的消息尚未公开
- 女主离开去执行秘密任务
- 章末钩子：远处传来异象

续写起始方式选择：
A. 直接接续第30章结尾，从异象开始
B. 时间跳跃（如3天后），从日常开始再引入新事件
C. 视角切换，先写其他角色的反应再回到主角
D. 自定义

❓ 你想从哪里开始续写？
```

### 决策点 ⑨：首章方向选择

**时机**：Phase 8 生成首章细纲后

**交互方式**：展示首章的3种可能方向

```
📖 第31章方向选择：

A. 紧张开局：异象→危机→主角被迫出手
   适合：提速线，快速进入冲突
   
B. 悬念开局：日常中暗藏不安→突然揭示
   适合：稳妥线，先铺垫再爆发
   
C. 反差开局：庆功的喜悦vs暗涌的危机
   适合：爆点线，情绪反差大

❓ 你想用哪种开局？
```

### 决策点 ⑩：Sample Chapter 讨论

**时机**：Sample Chapter 生成后

**交互方式**：不只问"行不行"，而是具体讨论

```
✏️ 试写片段（500字）：

[试写内容]

风格对比：
- 句式节奏：与原作接近 ✅
- 对话风格：偏正式，原作更口语化 ⚠️
- 情绪衔接：自然承接上一章 ✅

❓ 你觉得这个风格怎么样？哪里需要调整？
```

**选项**：
- A. 风格可以，继续写完整章节
- B. 对话太正式了，更口语化
- C. 节奏太慢，加快
- D. 完全不对，重新试写

### 决策点 ⑪：每章关键情节点确认

**时机**：Phase 10 每章正文生成前

**交互方式**：展示本章要写的关键点，让用户确认

```
📝 第31章即将生成，本章关键点：

1. 庆功宴被神秘来客打断
2. 来客揭示主角身世的一角
3. 主角情绪从喜悦转为震惊
4. 章末：来客留下一枚令牌后消失

❓ 这些情节点可以吗？有没有想加/改/删的？
```

**选项**：
- A. 可以，开始写
- B. 想调整某些点
- C. 这章先不写，跳过
- D. 想先讨论一下某个点

### 决策点 ⑫：审核结果讨论

**时机**：Phase 11 一致性审核和 Phase 12 润色评分后

**交互方式**：不只给评分，还讨论具体问题

```
📊 第31章审核结果：

一致性评分：8.2/10
风格指纹偏离：2项超阈值（对话占比偏低、情绪描写偏多）

主要问题：
1. 对话占比28%（原作35%），建议增加1-2段对话
2. 主角震惊后的反应偏内心独白，原作习惯用动作展示

❓ 你想怎么处理？
A. 按建议润色
B. 我觉得对话占比这样就行
C. 情绪描写我想保留，这是这章需要的
D. 重新写这一章
```

## 三、交互模式设计

### 两种交互模式

| 模式 | 特点 | 适合用户 |
|------|------|---------|
| 🗨️ 讨论模式（默认） | 每个决策点都展示选项+讨论 | 想深度参与方向的作者 |
| ⚡ 快速模式 | 只在关键决策点暂停，其余自动推进 | 想快速出稿的作者 |

**切换方式**：用户可在任意时刻说"切换到快速模式"/"切换到讨论模式"

**快速模式下的必停点**（不可跳过）：
- ④ 路线选择
- ⑥ 不可改动清单确认
- ⑩ Sample Chapter 确认
- ⑪ 首章关键情节点确认

### 交互格式规范

每个决策点的交互必须包含：
1. **当前状态摘要**：用 2-3 句话说明"我们现在在哪、在做什么"
2. **选项列表**：2-4 个选项 + 自定义入口
3. **推荐标注**：AI 推荐的选项标注"（推荐）"
4. **上下文提示**：为什么给出这些选项
