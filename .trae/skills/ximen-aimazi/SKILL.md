# 网文创作助手

## 技能概述

**核心价值**：结构化创作流程 + 深度写作技法 + 去AI味，三位一体
**适用场景**：长篇网文、爽文、玄幻、都市、仙侠、重生、系统流、同人
**语言**：中文（默认）

## 触发条件

用户提到以下关键词时自动触发：
- "写小说""长篇小说""小说创作""开书"
- "爽文""玄幻""都市""仙侠""穿越重生""系统流"
- "小说大纲""人物设定""世界观""细纲"
- "帮我写小说""创作灵感""去AI味""润色"

---

## 核心哲学

1. **大纲是地图，不是牢笼** — 粗到能看见全局，细到能坐下就写
2. **爽点密度决定存亡** — 每3000-5000字必须有一个让读者"爽"的情绪节点
3. **日更是底线，质量是上限** — 稳定产出比单章完美更重要
4. **先写30章，再谈其他的** — 大纲不需要完美，写出来自然发现方向
5. **网文写作是工程，不是灵感** — 靠工程可以写200万字，靠灵感不行

## Phase 1：灵感生成

**目标**：确定小说基本方向，生成3套完整创意方案

### 用户输入示例

用户可能只给一句话：
- "写一个都市修仙的爽文"
- "重生回高中逆袭成商业大亨"
- "废柴少年获得系统后一路碾压"

### 提示词自动完善

按以下8维度自动补全：

```
1. 题材定位    → 主类型 + 子类型
2. 世界观设定  → 力量体系、社会规则、时代背景
3. 主角人设    → 初始身份、性格、金手指
4. 核心冲突    → 主线矛盾 + 前3章即时冲突
5. 爽点设计    → 打脸节奏、升级频率、装逼方式
6. 节奏规划    → 每N章小高潮、每M章大高潮
7. 配角框架    → 对手/盟友/红颜各至少1人
8. 开篇钩子    → 第一章用什么抓住读者
```

### 快速问卷（可跳过）

如用户方向模糊：

```
1. 题材：[玄幻/都市/仙侠/其他]
2. 核心卖点：[系统流/穿越重生/逆袭打脸/其他]
3. 目标字数：[30万/50万/100万+]
4. 对标书：[有/无]
```

### 对标上下文加载

如果用户提到对标书，或工作目录下存在 `对标/` 目录：
1. 检查对标书的 `拆文报告.md` 是否存在
2. 如存在，读取核心发现作为参考上下文
3. 如不存在，提示用户是否需要先做拆文分析

### 输出

1. **3个备选书名**：直白有冲突，吸引目标读者
2. **一句话简介**：主角+困境+金手指+目标
3. **核心冲突**：主角目标+最大阻碍+对抗方式
4. **核心爽点**：每章能让读者获得的情感满足

## Phase 2：世界观构建

**目标**：基于选定题材构建完整、自洽的世界观

### 快速构建

```
1. 时代背景：[现代都市/古代仙侠/未来星际/异世界]
2. 力量体系：[武道等级/魔法元素/异能觉醒/系统规则]
3. 社会结构：[宗门统治/家族势力/城邦自治/联邦制]
4. 核心规则：[弱肉强食/实力至上/炼丹师稀缺]
```

### 详细构建（可选）

| 要素 | 内容 |
|------|------|
| 基础框架 | 时代背景、地域分布、社会结构 |
| 力量体系 | 等级详细、修炼方式、突破条件 |
| 势力地图 | 国家、宗门、家族的分布与关系 |
| 禁忌常识 | 社会潜规则、绝对禁忌 |
| 关键场景 | 主要城市、秘境、战场 |

### 世界观设计要点

- 力量体系必须有明确边界和代价
- 世界观展开跟剧情推进绑定（不搞说明文）
- 日常细节比宏大设定更让世界"活"
- 每5章必须有一次世界观层面的新信息

## Phase 3：角色塑造

**目标**：创建立体、有深度的角色

### 主角卡

```
姓名：
身份标签：（废柴大学生/前特种兵/落魄皇子）
外貌特征：（3-5个关键词，要有记忆点）
性格关键词：（3-5个，必须有矛盾面）
核心目标：（全书终点想达成什么）
核心动机：（为什么一定要达成——情感驱动，不是理性驱动）
致命弱点：（让角色犯错的性格缺陷）
金手指/系统：[具体能力和规则+限制]
口头禅/标志动作：
```

**设计要点**：
- 动机必须是情感层面的（"为母亲复仇"优于"要成为最强"）
- 弱点必须会在关键情节导致主角犯错
- 主角行为三必须：可理解、可共鸣、可接受

### 配角卡

```
姓名：
角色功能：（导师/盟友/情报源/牺牲品/镜像对照）
与主角关系：
核心特质：（1-2个关键词）
标志性特征：（一句话让读者记住）
退场方式：（何时/如何退出故事）
```

### 反派层级

| 层级 | 跨度 | 设计要点 |
|------|------|---------|
| 小反派 | 1-5章 | 1-2个鲜明特征，出场快退场干脆 |
| 中等反派 | 10-30章 | 有动机+有手段+有逼格，至少赢主角一次 |
| 大弧Boss | 一卷或数卷 | 完整人物弧线，有理念冲突，有让读者"恨不起来"的侧面 |
| 最终Boss | 全书 | 从第一章就有伏笔，代表故事主题的反面 |

**反派铁律**：反派的智商/实力决定主角的含金量。弱反派=弱主角=无聊故事。

### 人物关系类型

| 类型 | 功能 | 示例 |
|------|------|------|
| 冲突型 | 制造张力，推动情节 | 宿敌、竞争对手 |
| 联盟型 | 提供助力，制造羁绊 | 战友、师徒 |
| 亲密型 | 制造软肋，提供情感支点 | 恋人、家人 |
| 权威型 | 制造压力，限制行动 | 师父、老板 |

### 三层标签反差人设法

| 层级 | 名称 | 示例 |
|------|------|------|
| 第一层 | 身份标签（外界看到的） | 豪门弃妇 |
| 第二层 | 表现标签（展现给外界的） | 隐忍不发 |
| 第三层 | 内核标签（真正的内心） | 冷静、步步为营 |

三层之间必须有反差，反差=立体感。

### 动机链

```
起因（Cause）    → 角色经历了什么
意图（Intent）   → 角色想做什么
约束（Constraint）→ 什么在阻碍他
风险（Risk）     → 失败的代价是什么
```

### 输出文件

- `设定/角色/{角色名}.md`：每个角色一个文件

## Phase 4：创作参数配置（交互式）

**目标**：在生成大纲前，让用户确认关键创作参数

### 必问参数

**使用 `AskUserQuestion` 工具一次性询问以下参数**：

#### 参数1：章节数量

```
50章（短篇/试水）/ 100章（标准）/ 200章（长篇）/ 500章（超长篇）/ 自定义
```

#### 参数2：女角色数量

```
1个（唯一女主）/ 2个（双女主）/ 3个（经典三女主）/ 5个（后宫向）/ 自定义
```

#### 参数3：擦边等级

```
无（纯剧情）/ 少量（偶尔暧昧）/ 中量（定期安排）/ 大量（频繁出现）
```

| 等级 | 频率 | 安排 |
|------|------|------|
| 无 | 0% | 不安排 |
| 少量 | 5-10% | 每10-20章1次暧昧互动 |
| 中量 | 15-25% | 每4-6章1次擦边情节 |
| 大量 | 30-40% | 每2-3章1次擦边情节 |

### 参数确认流程

```
1. 一次性询问所有参数
2. 如选择了擦边（非"无"）→ 询问擦边描写风格
   选项：A含蓄/B直白/C文艺/D暴力美学/E自定义
3. 将所有参数写入 设定/创作参数.md
4. 根据参数调整后续流程
```

### 5种预设文风

| 文风 | 特点 | 适用题材 |
|------|------|---------|
| A 热血燃向 | 短句密集、节奏快、情绪高亢 | 玄幻、高武、升级流 |
| B 冷峻硬核 | 白描为主、克制内敛、细节精准 | 悬疑、规则怪谈、末世 |
| C 轻松日常 | 口语化、生活感、节奏舒缓 | 都市、种田、御兽 |
| D 古韵仙侠 | 半文半白、意境优先、节奏沉稳 | 仙侠、古言、玄侠 |
| E 幽默反套路 | 反差感、吐槽密度高、节奏跳跃 | 搞笑修仙、法治修仙、脑洞文 |

用户也可选择"自定义文风"，提供参考作品或风格描述。

3. 保存到 设定/原著资料.md

| 类型 | 约束 |
|------|------|

## Phase 6：大纲生成

**目标**：制定完整的小说结构

### 大纲创建五步法

1. **确定高潮剧情** — 最大冲突规模、最多出场人物、最强情绪波动
2. **确定单元剧** — 主角如何使用金手指获得爽点，相邻单元剧不能使用相同金手指逻辑
3. **故事线组织** — 同时管理8条线：地图/阵营/人物/金手指/世界观/矛盾/收集/感情
4. **确定开篇舞台** — 情节钩子→异常状态→金手指→矛盾/欲望→目标建立
5. **确定收尾** — 解决主线+额外奖励+角色结局+节奏放缓+下卷铺垫

### 卷级大纲结构

```markdown
# 《小说名》大纲

## 基本信息
- 题材：
- 预计章节数：
- 每章字数：约3000字

## 力量/等级体系

## 卷级大纲
### 第一卷：[卷名]（第1-N章）
- 功能：铺垫/起步/第一个大爽点
- 核心事件：一句话
- 起始状态 → 结束状态：主角从A变成B
- 情绪弧线：[V形/倒V形/W形/递进/延迟满足/急转]

### 第二卷：[卷名]
...
```

### 三幕式节奏

| 阶段 | 章节占比 | 内容 |
|------|---------|------|
| 第一幕 | 1-20% | 建立常态→打破常态→接受使命 |
| 第二幕 | 20-75% | 试炼成长→中点转折→至暗时刻 |
| 第三幕 | 75-100% | 最终准备→高潮对决→结局余韵 |

### 关键节点（以100章为基准，按总章节数等比缩放）

| 节点 | 比例 | 内容 |
|------|------|------|
| 开篇钩子 | 1% | 必须吸引读者 |
| 第一个高潮 | 10% | 确立故事方向 |
| 中点转折 | 50% | 提升故事格局 |
| 至暗时刻 | 65% | 情感最低点 |
| 最终高潮 | 85% | 所有矛盾爆发 |

### 升级感三步设计法

1. **列出起点**：主角当前身份、阶层、地位、资源
2. **列出终点**：本卷结束后的收获和提升
3. **反向设置情绪缺口**：针对终点的收获，在前文铺垫需求

### 输出文件

- `大纲/大纲.md`：全书卷级结构
- `大纲/卷纲_第X卷.md`：每卷的爽点节奏+情绪弧线+人物弧线+伏笔+反转

## Phase 7：细纲规划

**目标**：把全书细纲改造成可迭代的批次工作流

### 规划原则

1. **先卷级，后批次，再到单章**
2. **默认只做当前批次+下一批缓冲**
3. **按风险调整批次大小**
4. **区分硬锚点与软空间**
5. **先定义冻结条件，再开始细纲**

### 推荐批次大小

| 场景 | 建议每批章数 |
|------|-------------|
| 开篇前20章 | 3-5章 |
| 稳定推进期 | 5-8章 |
| 大高潮/换地图/换卷 | 2-4章 |

> 单批不建议超过10章。

### 规划内容

```
1. 按卷整理核心冲突、卷末状态、必须兑现的大事件
2. 把每卷拆成多个"细纲批次"
3. 为每个批次定义：目标、爽点、章末钩子、风险点
4. 标记硬锚点和可调整空间
5. 约定冻结条件、人工介入点和返修触发条件
```

### 输出文件

- `大纲/细纲.md`：细纲主控文件
- `追踪/伏笔.md`：伏笔状态表
- `追踪/时间线.md`：故事时间线

## Phase 8：生成细纲

**目标**：只生成当前批次的详细细纲，为后续返修预留接口

### 生成前必读

- `大纲/大纲.md`
- `大纲/卷纲_第X卷.md`
- `设定/` 活跃部分
- `追踪/伏笔.md`（如已存在）

### 生成规则

1. **只生成当前批次**
2. **带着上下文生成** — 至少读取前一个已冻结批次+当前卷目标
3. **每章必须有功能** — 推进了什么、兑现了什么、留下了什么
4. **给返修留接口** — 标出可微调项和硬锚点
5. **批次生成完成后先不写正文** — 必须先过Phase 9审核并冻结

### 批次细纲内容

每章细纲包含：

| 项目 | 内容 |
|------|------|
| 章节定位 | 章节类型、本章任务、与上章衔接 |
| 兑现内容 | 本章交付给读者的爽点、信息或关系变化 |
| 情节点 | 3-5个关键推进点 |
| 人物变化 | 出场人物、状态变化、关系变化 |
| 世界变化 | 地点、资源、势力、规则变化 |
| 伏笔管理 | 本章埋设/回收的伏笔 |
| 章末钩子 | 留给下一章的悬念（从钩子13式中选择） |
| 返修接口 | 可微调项/不可轻改项/风险提示 |

### 输出文件

- `大纲/细纲.md`（主控版）
- `大纲/批次细纲/第001-005章_v1.md`（当前批次草稿）

## Phase 9：逻辑审核与人工介入

**目标**：审核当前批次，给出可选方案，让用户介入后决定冻结/小修/回退

### 审核维度

| 维度 | 检查项 |
|------|--------|
| 时间线 | 章节衔接、时间跳跃、成长速度 |
| 空间线 | 地点转移、行动距离、地图切换成本 |
| 人物线 | 存活状态、等级匹配、关系变化、动机连贯 |
| 设定线 | 力量体系、物品来源、社会规则、系统边界 |
| 伏笔线 | 埋设、回收时机、后续可承接性 |
| 读感线 | 章节功能是否清楚、爽点是否兑现、章末钩子是否有效 |

### 审核流程

```
1. 读取：上一批已冻结细纲 + 当前批次细纲 + 当前卷目标
2. 逐章检查当前批次
3. 跨章节检查：至少回看前3-5章与下一批目标
4. 记录问题并判断级别：微调/当前批重做/回改上一批/回改大纲
5. 生成2-4个方案+1个自定义选项，进入人工介入节点
6. 用户选择后修改并重新审核，直到满足冻结条件
7. 更新追踪文件
```

### 冻结条件

- 无硬逻辑矛盾
- 本批次承诺的爽点已兑现
- 本批次结尾能自然牵引下一批
- 人工介入节点已完成

## Phase 10：正文章节

**目标**：按审核通过的细纲生成高质量正文

### 写作前读取

1. `正文/第{N-1}章_*.md` — 上一章正文
2. `大纲/批次细纲/` — 当前批次细纲（含本章）
3. `追踪/伏笔.md` — 待回收伏笔
4. `设定/角色/{相关角色}.md` — 本章涉及角色
5. `设定/题材定位.md` — 核心梗与对标参考

### 开头设计（黄金一章法则）

- 不再有"黄金三章"的奢侈，现在是黄金一章
- 从故事最精彩、最有冲突的地方写起
- 主角要么身处危机，要么处于令人羡慕的状态，要么被丢入完全陌生环境

**三种开局模板**：

| 模板 | 结构 | 适用 |
|------|------|------|
| 困境切入型 | 不利处境→压迫/危机→金手指激活→初步解决但引出更大问题 | 逆袭流、重生文 |
| 优势切入型 | 已有独特优势但被低估→优势意外暴露→周围人认知刷新→暴露引来更大麻烦 | 扮猪吃虎、无敌流 |
| 悬念切入型 | 抛出反常事实→层层剥开→颠覆认知 | 悬疑、规则怪谈 |

**开篇信息释放优先级**：危机感 > 人设 > 金手指暗示 > 世界观

### 写作技法提醒

| 场景 | 技巧 |
|------|------|
| 开篇500字 | 必须有钩子，不能从天气/风景开始 |
| 对话 | 推进剧情或揭示性格，不能只为了凑字数 |
| 打斗 | 写策略和反转，不写"你一拳我一脚" |
| 日常 | 要有人物互动和伏笔，不能只是"吃饭睡觉" |
| 爽点释放 | 铺垫要充分、释放要干脆 |
| 章尾 | 每章结尾都要有让读者想翻下一页的东西 |

| 节奏 | 字数区间 | 内容密度 |
|------|----------|----------|
| 高速推进 | 2000-3000字/章 | 每章一个明确事件 |
| 正常节奏 | 3000-4000字/章 | 主线+少量副线 |
| 舒缓铺垫 | 3000-4000字/章 | 人物互动+伏笔 |
| 高潮爆发 | 2000-3000字/章 | 集中释放、不拖沓 |

### 去AI味写作规范（正文必遵）

正文写作时必须遵循以下去AI味规范，从源头避免AI痕迹：

#### 核心规则

1. **段落不超过3句** — 超过就拆
2. **动作+对话+情绪反应循环推进** — 不要单写心理活动超过2段
3. **短句优先** — 打斗3-8字，对话口语化，日常8-15字
4. **口语化表达** — 允许俚语粗话，对话不要书面语
5. **Show Don't Tell** — 用行为代替形容词，用细节代替总结

#### 禁用词（出现即替换）

| 类别 | 禁用 | 替换为 |
|------|------|--------|
| 情态 | 仿佛、好像、犹如、宛若、一丝、一抹 | 删掉或具体描写 |
| 动作 | 深吸一口气、缓缓、不禁、微微、轻轻、淡淡 | 删掉或具体动作 |
| 表情 | 眼中闪过、嘴角勾起、眉头微皱、瞳孔微缩 | 具体表情/动作 |
| 心理 | 心中一动、心头一震、心下了然、不由得 | 用动作展示思考 |
| 判断 | 不容置疑、显而易见、毫无疑问 | 删掉 |
| 过渡 | 突然、瞬间、不由自主、情不自禁 | 删掉或改写 |

#### 章末禁止

- ❌ 总结性感悟（"他终于明白了……"）
- ❌ 升华式感叹（"这一夜，注定无人入眠"）
- ❌ 哲理式收尾（"人生就是这样……"）
- ❌ 伏笔式预告（"他不知道的是，更大的风暴即将来临"）
- ✅ 用动作、对话或悬念收束

#### 比喻禁令

绝对禁止：像、就像、好像、如同、仿佛、宛如、犹如、好似

```
❌ 他像箭一样冲出去
✅ 他冲了出去
```

## Phase 12：润色与评分

**目标**：对正文做最终润色，确保质量达标

### 去AI味6门禁（润色必过）

#### 门禁A：禁用词替换

对照禁用词表逐项检查，用具体动作/细节描写替换。

```
❌ "眼中闪过一丝不易察觉的悲伤" → ✅ "他垂下眼"
❌ "深吸一口气" → ✅ "胸口起伏了一下"（或直接删掉）
❌ "嘴角勾起一抹冷笑" → ✅ "他笑了一下，没到眼底"
```

#### 门禁B：句式去套路

| 句式 | 问题 | 替代 |
|------|------|------|
| "…，带着…" | 万能状语 | 用独立短句或动作描写 |
| "像XX一样" | 陈词滥调 | 换比喻或直接白描 |
| "他/她知道…" | 直接告诉 | 用行为展示认知 |
| "XX说道" | 机械标签 | 用动作替代"说道" |
| "仿佛/犹如/宛若" | 文言腔 | 口语化或白描 |

#### 门禁C：心理描写外化

```
"他很紧张" → "他的手在抖"
"她很愤怒" → "她一把掀翻了桌子"
"他很害怕" → "他的腿在发抖，几乎站不稳"
"她很伤心" → "她转过身去，肩膀微微颤动"
```

#### 门禁D：节奏打碎

- 打断连续排比句（保留1-2个，删掉其余）
- 长句拆短句
- 偶尔用不完整句（口语感）
- 段落长短交错

#### 门禁E：对话去腔调

- 加入口语化表达（"嗯""哦""行吧"）
- 适当打断对话（角色可以答非所问）
- 用动作穿插对话
- 删掉解释性对话

#### 门禁F：结尾去升华

- 删掉总结性语句
- 用动作/场景收尾，不要用感慨收尾
- 如果结尾有"他知道…""这一刻…"→ 基本可以删

### 去AI味三遍法

| AI味程度 | 策略 |
|----------|------|
| 轻度 | 只做Pass 1（去泛化） |
| 中度 | Pass 1 + Pass 2（去书面化） |
| 重度 | 完整三遍 + 重点段落重写 |

**Pass 1：去泛化** — 抽象情绪→具体动作，假深度句→删，工整对比→打散，装饰性形容词→白描

**Pass 2：去书面化** — 分析性用词→日常表达，抽象名词→直接说事，体制内用语→删

**Pass 3：回人味** — 感官细节，角色说话区分，节奏变化，场景特有记忆点

## 期待感管理六法则

1. **期待最大化定律** — 将要满足却还没满足时张力最高
2. **信息顺序操纵** — 先植入什么信息，读者就先入为主往哪想
3. **期待递增法** — 在基底期待上不断添加更具体的需求缺口
4. **断期待禁止** — 下一个期待立起来之前，绝不能结束当前期待
5. **下行情节安全原则** — 锅都是别人的，功都是主角的
6. **爽点递增对比** — 连续爽点逐级递增（影响范围/揭示深度/身份落差）

只借结构、节奏、冲突处理，不逐句模仿。

### 模式B：半部续写/卡文救援

查阅 `references/continuation-engine.md`，执行六步续写引擎：

1. **作品DNA拆解** — 类型承诺、主角驱动力、主线问题、文风指纹
2. **事实校准** — 对照现有正文确认底层事实
3. **卡点诊断** — 结构卡/人物卡/资源卡/信息卡/情绪卡
4. **三条续写路线** — 稳妥线/提速线/爆点线
5. **拆成3-10章节拍** — 落到章节可执行层面
6. **试写低风险开口** — 下一章开头500-1000字

### 模式C：AI编辑部流水线

查阅 `references/editorial-pipeline.md`，按岗位顺序运行：

| 岗位 | 职责 |
|------|------|
| 题材参谋 | 评估方向、卖点、平台感 |
| 架构编辑 | 定世界观、人物关系、分卷大纲 |
| 章节执行 | 按细纲写正文 |
| 毒舌审稿 | 找问题、标风险、抓水文 |
| 改稿编辑 | 根据报告修订 |

### 模式D：深度审稿

查阅 `references/advanced-audit.md`，排查长篇中后段最容易忽略的人设、资源、伏笔和追读风险。
