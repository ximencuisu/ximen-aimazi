# CP1 反向解析操作指南

> 何时加载：Continuation_Mode 激活后，进入 CP1 阶段时。

## 概述

CP1 是续写流程的第一个准备阶段，目标是从用户提供的已有正文中系统化提取作品DNA、风格指纹、人物状态、伏笔状态、世界观规则，让 AI 真正"读懂"原作而不是凭感觉续写。

---

## 输入素材处理

### 素材类型

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| 正文章节文件 | 用户提供的 .md 或 .txt 文件 | 逐文件读取，按文件名排序确定章节顺序 |
| 纯文本粘贴 | 用户直接粘贴的正文内容 | 合并为连续文本，按自然分段识别章节 |
| 混合素材 | 原作 + 用户自己续写半截的内容 | 识别分界点，分别标记来源 |

### 样本量检查

| 字数 | 处理方式 |
|------|---------|
| < 5000 字 | 提示"原文样本量偏低，风格指纹置信度可能不足"，询问是否继续 |
| 5000-30000 字 | 正常处理，风格指纹置信度中等 |
| > 30000 字 | 正常处理，风格指纹置信度较高 |

## 产出物提取方法

### 1. 作品DNA提取

从正文中提取以下五项：

| 项目 | 提取方法 | 示例 |
|------|---------|------|
| 类型承诺 | 识别题材类型和读者期待 | "玄幻升级流，承诺读者看主角从废材到巅峰" |
| 主角驱动力 | 提取主角的核心目标和情感动机 | "为父复仇→证明自己→守护身边人" |
| 主线问题 | 识别全书要解决的核心矛盾 | "主角能否在三年之约中战胜对手" |
| 副线结构 | 梳理情感线、势力线、悬念线等 | "感情线：与女主从敌对到并肩；势力线：从散修到宗门核心" |
| 读者爽点契约 | 归纳读者追这本书期待什么爽感 | "装逼打脸+升级碾压+身份揭秘" |

**矛盾检测**：如果正文中存在前后矛盾的设定（如角色能力前后不一致、时间线冲突），在 `设定/作品DNA.md` 中列出矛盾清单并提示用户人工裁决。

### 2. 风格指纹提取

#### 九项指标计算方法

**平均句长**：
- 统计方法：以句号、问号、感叹号为分隔，计算每句字符数的平均值
- 示例：平均句长 18.5 字

**句长分布区间**：
- 统计方法：将句子按长度分为短句（≤10字）、中句（11-25字）、长句（>25字），计算比例
- 示例：短句 30% / 中句 50% / 长句 20%

**平均段长**：
- 统计方法：以空行为分隔，计算每段行数的平均值
- 示例：平均 2.3 行/段

**对话占比**：
- 统计方法：以引号标记的对话行数 / 总行数
- 示例：对话占比 35%

**高频意象前20**：
- 统计方法：提取名词性意象词（非人名、地名），按频率排序取前20
- 示例：剑气、灵石、丹药、阵法、符箓...

**高频动词前20**：
- 统计方法：提取动作动词，按频率排序取前20
- 示例：斩、破、凝、催、掠...

**句式偏好**：
- 统计方法：统计陈述句/疑问句/感叹句/倒装句的比例
- 示例：陈述 70% / 疑问 10% / 感叹 15% / 倒装 5%

**动作-对话-情绪比例**：
- 统计方法：将每段文字分类为动作描写、对话、情绪/心理描写，计算比例
- 示例：动作 40% / 对话 35% / 情绪 25%

**原作回避表达倾向**：
- 统计方法：对比 `references/banned-words.md` 中的禁用词表，识别原作中从未出现但同类题材常见的表达
- 示例：原作从不使用"仿佛""宛如"，则标记为回避表达

#### 跨作者接力（F）额外提取

- 标志性句式：识别原作者反复使用的特殊句式结构
- 叙述视角偏好：第一人称/第三人称有限/第三人称客观/跳跃视角
- 视角切换规律：何时切换视角、切换频率

### 3. 角色卡提取

每个出现过的有名有姓的角色独立成卡：

```markdown
## {角色名}

- **外貌特征**：从正文描写中提取关键词
- **性格关键词**：3-5个，须有矛盾面
- **关系网络**：与其他角色的关系
- **当前状态**：最后一次出场时的状态
- **当前所在地**：最后一次出场的地点
- **持有资源**：已知拥有的物品、能力、身份
- **出场章节**：首次出场 ~ 最后出场
```

### 4. 伏笔表提取

```markdown
| 伏笔编号 | 伏笔内容 | 埋设章节 | 当前状态 | 相关角色 | 回收建议 |
|----------|---------|---------|---------|---------|---------|
| F001 | 主角身世之谜 | 第3章 | 已埋未收 | 主角、神秘老者 | 建议在第25-30章部分揭示 |
```

状态分类：
- 已埋未收：已埋设但尚未有任何回收迹象
- 部分回收：已有部分揭示但未完全解决
- 已回收：已完全揭示或解决

### 5. 世界观提取

**背景设定**：从正文中归纳时代背景、社会结构、核心规则

**力量体系**：从正文中归纳力量等级、边界、代价

提取方法：
1. 找出所有涉及力量/等级/境界的段落
2. 归纳等级体系（如练气→筑基→金丹）
3. 识别每个等级的能力边界和突破代价
4. 标注信息来源章节

## .learnings/ 回填流程

### 回填顺序

1. `STORY_BIBLE.md` — 世界观核心规则
2. `CHARACTERS.md` — 角色档案
3. `LOCATIONS.md` — 地点信息
4. `PLOT_POINTS.md` — 关键情节
5. `SUSPENSE.md` — 伏笔状态
6. `RESOURCES.md` — 物品/金钱
7. `SUBPLOTS.md` — 子剧情
8. `EMOTIONS.md` — 情感发展

### 裁决规则

当 `.learnings/` 档案在 CP1 之前已存在内容时：

| 情况 | 处理方式 |
|------|---------|
| 原有内容与原作提取一致 | 保留原值，标注来源 |
| 原有内容与原作提取矛盾 | 列出差异，用户裁决"保留原值/用原作值覆盖/合并" |
| 原有内容有但原作未提及 | 保留原值，标注"原作未验证" |
| 原作有但原有内容缺失 | 追加原作提取值 |

### Metadata 标注

每条写入项末尾追加：
```
[来源: 原作提取 | 提取时间: YYYY-MM-DD HH:mm]
```
