# 文风预设库

本文档不再承担“当前项目激活文风”的职责，而是作为可复用的文风预设库使用。

长篇网络小说的当前生效文风，应写入 `memory/project_style.md`。
推荐流程：

1. 从本文件选择 `1 个基础文风`
2. 如确有需要，额外借 `1 个辅助元素`
3. 如需作家技法参考，再叠加 `1 位主参考 + 1 位辅参考`
4. 将最后得到的长期写法写入 `memory/project_style.md`

---

## 长篇使用原则

- 整本书固定 1 个基础文风，不在每章重新选
- 辅助文风建议不超过 30%，避免长期连载时文风打架
- 作家技法参考只借节奏、结构、钩子、冲突和人物处理，不做句子级模仿
- 只在开书、换卷、平台变化或明显风格漂移时调整

---

## 作家技法参考使用规则

> 作家技法参考是叠加在基础文风之上的"写法借鉴"，不是换风格、不是抄句子。
> 核心约束：**只借节奏、结构、钩子、冲突和人物处理，不做句子级模仿。**

### 选择流程

1. **基础文风先行**：先从「文风预设」选定 1 个基础文风，写入 `memory/project_style.md`。
2. **主参考**：从 `references/author-style-guide.md` 选择 1 位与基础文风匹配的作家作为主参考，提取其可迁移技法。
3. **辅参考**（可选）：如需叠加，再选 1 位辅参考，占比不超过 30%，只吸收其差异化长处。
4. **写入项目文风圣经**：主参考、辅参考、使用比例、可迁移技法、禁止误用项一并写入 `memory/project_style.md` 的「作家技法参考」段。
5. **上限**：最多 `1 位主参考 + 1 位辅参考`，不允许多作家混搭导致文风打架。

### 约束清单

- [ ] 只借**节奏**（爽点密度、推进速度、张弛节奏），不抄具体桥段
- [ ] 只借**结构**（章节功能、卷级闭环、伏笔布局），不照搬章节划分
- [ ] 只借**钩子**（章首/章末钩子类型与强度），不复制原句悬念
- [ ] 只借**冲突**（矛盾升级方式、对抗设计），不抄袭具体冲突桥段
- [ ] 只借**人物处理**（人设立体度、动机推进、关系张力），不照搬角色设定
- [ ] **禁止句子级模仿**：不逐句仿写、不复制专属名词体系、不照搬代表桥段
- [ ] **禁止风格漂移**：技法只作用于上述 5 个维度，基础文风保持不变

### 与 author-style-guide.md 的衔接

- 候选作家及其可迁移技法、禁止误用项详见 `references/author-style-guide.md`。
- 本文件只规定"如何使用"作家技法参考的流程与约束；具体作家技法提炼在 `author-style-guide.md` 中查阅。
- 选定后在 `memory/project_style.md` 记录：主参考 / 辅参考 / 使用比例 / 可迁移技法 / 禁止误用，正文生成前必读。
- 若用户指定作者风格，按 `SKILL.md` "可选增强模式 → 作家技法参考" 触发，读取 `references/author-style-guide.md`。

### 【文风A】热血爽文风

```markdown
## 核心特点
- 节奏快，爽点密集
- 打脸干脆利落，不留余地
- 升级突破描写要燃
- 主角气场强大，碾压对手

## 句式要求
- 短句为主，干净利落
- 动作描写要快：「他出手了。」而不是「他缓缓抬起手，准备出手。」
- 对话简洁有力，不废话
- 避免长段心理描写

## 禁止事项
- 禁止拖沓的环境描写（超过2句）
- 禁止过多的旁白解释
- 禁止主角犹豫不决
- 禁止反派废话连篇

## 爽点节奏
- 每500字至少一个爽点
- 打脸要彻底，不要给反派喘息机会
- 突破要震撼，要有围观群众惊叹
```

### 【文风B】都市现实风

```markdown
## 核心特点
- 贴近现实，有烟火气
- 对话自然，像真人说话
- 细节真实，有生活感
- 人物有缺点，不完美

## 句式要求
- 口语化表达
- 适当使用方言或俚语
- 心理活动要真实，不矫情
- 场景描写要有画面感

## 禁止事项
- 禁止过于中二的台词
- 禁止不切实际的金手指设定
- 禁止脸谱化的反派
- 禁止过度装逼

## 情感节奏
- 感情线要循序渐进
- 冲突要有合理动机
- 转折要有铺垫
```

---

### 【文风C】古风仙侠风

```markdown
## 核心特点
- 文字有古韵，但不晦涩
- 仙气飘飘，意境深远
- 战斗场面要写意
- 人物气质出尘

## 句式要求
- 四字词语适当使用
- 可以用对偶、排比
- 描写要有画面感和意境
- 对话可以文雅一些

## 禁止事项
- 禁止过于现代化的表达
- 禁止网络流行语
- 禁止过于血腥暴力的描写
- 禁止完全文言文化（影响阅读）

## 特色元素
- 法宝、丹药、阵法的描写要古雅
- 境界突破要有天象异动
- 人物名字要有仙气
```

### 【文风D】悬疑惊悚风

```markdown
## 核心特点
- 氛围压抑，悬念迭起
- 节奏张弛有度
- 细节暗示，伏笔密布
- 反转出人意料

## 句式要求
- 多用短句制造紧张感
- 环境描写要阴暗压抑
- 心理描写要细思极恐
- 对话要有潜台词

## 禁止事项
- 禁止过早揭示真相
- 禁止无意义的惊吓（jump scare）
- 禁止逻辑漏洞
- 禁止主角过于被动

## 悬念节奏
- 每章结尾必须有悬念
- 线索要分散埋设
- 真相要层层揭开
```

### 【文风E】轻松日常风

```markdown
## 核心特点
- 轻松幽默，读起来愉快
- 日常琐碎但有趣
- 角色互动有爱
- 没有太大压力

## 句式要求
- 口语化，像朋友聊天
- 可以用吐槽和玩梗
- 心理活动可以搞笑
- 对话要活泼

## 禁止事项
- 禁止过于沉重的剧情
- 禁止虐心情节
- 禁止复杂的政治斗争

- 剧情推进可以慢一些
- 重点放在人物互动
- 日常小事要有趣
```

### 【文风F】对比学习文风

> **核心理念**：不说你喜欢什么风格，直接给我看——一段你不想要的 + 一段你想要的，AI 自动提取可执行的写作规则。

```markdown
## 核心特点
- 基于用户提供的对比样本（AI原文 vs 用户修改文），自动提取风格差异
- 规则从实例中来，每条规则都有「反例 → 正例」对照
- 规则优先级高于泛化去AI味规则和题材风格模块
- 适用于：去AI味优化 / 个人风格固化 / 模仿特定作品/作者风格

## 使用流程
1. 在 `assets/STYLE-CONTRAST-TEMPLATE.md` 中贴入原文（不想要的）和修改文（想要的）
2. 要求 AI 按 `references/style-contrast-analysis.md` 的10个维度执行对比分析
3. AI 输出结构化规则写入 `memory/style_contrast_guide.md`
4. Phase 4 选择文风F，`memory/project_style.md` 的基础文风填入 F
5. 后续 Phase 10（正文）和 Phase 12（润色）强制执行对比规则

## 句式要求
- 由对比分析结果决定，遵循 `memory/style_contrast_guide.md` 中的具体规则

## 禁止事项
- 由对比分析结果决定，遵循 `memory/style_contrast_guide.md` 中每条规则的「禁止事项」

## 优先级
- 对比学习文风规则 > 泛化去AI味规则（`anti-ai-writing.md`）> 题材风格模块（`style-*.md`）
- 当 `memory/style_contrast_guide.md` 中的规则与其他参考冲突时，以对比规则为准
```

> **内建默认基线**：即使用户未提供对比样本，`memory/style_contrast_guide.md` 中已预置 12 条通用「去AI味写作规则」（基于玄幻/修仙网文常见 AI 味模式提取），开箱即可生效。

- 80% 主文风

## 使用说明

1. 从本文件挑选一个最接近项目需求的基础文风
2. 如果需要混搭，只额外拿 1 个辅助元素，不要大杂烩
3. 如果用户指定作者技法，再配合 `references/author-style-guide.md` 提炼可迁移技法
4. 将最终结果写入 `memory/project_style.md`
5. 正文、大纲、细纲生成时优先读取 `memory/project_style.md`
6. 本文件保留为“预设库”，不要把它当成当前项目状态文件反复改写

## 卷级微调建议

长篇连载如需调整风格，优先做“卷级微调”，而不是“章节级频繁换风格”：

```markdown
## 卷级微调

| 卷/阶段 | 保持不变 | 允许微调 |
|---------|----------|----------|
| 开篇期 | 主文风、句式、叙述距离 | 钩子强度更高 |
| 中段推进期 | 主文风、人物说话习惯 | 节奏更稳，信息量更大 |
| 大高潮卷 | 主文风、角色底色 | 冲突强度、句长、章末悬念更强 |
```

## 章节级覆盖原则

只有在特殊章节才建议临时覆盖，例如：

- 大高潮章
- 审讯/博弈章
- 恐怖氛围章
- 情感爆发章

而且覆盖只生效一次，用完就回归 `memory/project_style.md` 的主文风。
